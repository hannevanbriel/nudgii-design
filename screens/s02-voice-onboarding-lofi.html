Shell + layout
Rule Shell pattern
Progress bar: 40% fill, outside SafeArea, 3px height, full-width edge-to-edge.
No screen title: nudgii's opening bubble IS the screen entry point.
No back button: forward-only flow. iOS swipe-back gesture handles navigation.

Shell: SafeArea + Expanded(conv thread) + fixed input bar
Conversation pattern
Pattern Bubble anatomy
nudgii bubbles: left-aligned, white bg, bottom-left flat corner (12px 12px 12px 4px). Max-width 88%. Sender label "nudgii" (always lowercase) above first bubble in a sequence.

User bubbles: right-aligned, CTA purple bg (#9B7FD4), bottom-right flat corner (12px 12px 4px 12px). Max-width 84%.

Confirmed items: sage-lt (#EAF3DE) chips with checkmark icon. Appear inside nudgii's confirmation bubble after each turn.

Suggestion chips: surface bg, subtle border, update contextually after each turn. Only show items not yet added.

Continue bubble: purple-tinted background signals a transition moment. Contains "Show my schedule" (filled CTA chip) and "Add more first" (default chip).
Rule Input bar
Fixed at bottom, outside scroll. Three input modes in one bar:
  • Scan icon (left): PhosphorIcons.scan, 14px, for camera/AI scan input
  • Text input (center): placeholder "Say it, type it, or snap a photo..."
  • Mic button (right): 28px circle, CTA bg, Phosphor Microphone

Privacy note: always visible below the bar. "audio sent for transcription only · responds in text"

"or browse instead" link: right-aligned below input bar. Lets users switch to S-03 Browse without swiping back. Appears after 2+ items are confirmed.
Navigation decisions
Interaction Navigation paths
  • "Show my schedule" (CTA chip) navigates to S-04 Review
  • "Add more first" (default chip) keeps conversation open
  • "or browse instead" (link) navigates to S-03 Browse

Max 1 clarifying question per nudgii turn. nudgii never asks two questions in the same bubble. Single question keeps the conversation fast and unblocking (critical for Arne persona).

Max 1 minute per voice turn. Countdown appears at 50s remaining.
Conversation flow
Note Turn-by-turn flow
Turn Speaker Content
1 nudgii "What do you have at home?" + hint + suggestion chips (dishwasher, boiler, car, lawn)
2 user Items via voice, text, or scan (e.g. "A dishwasher and a cv boiler.")
3 nudgii Confirms with sage chips + asks for more categories ("Anything else? A car, garden, subscriptions?")
4 user More items (e.g. "Two cars and an olive tree.")
5 nudgii Confirms + "Ready to see your schedule?" with CTA chip ("Show my schedule") + fallback ("Add more first")
Note 4 screen states (hi-fi shows all)
  • State 1: Fresh start. nudgii opening bubble + suggestion chips + idle input bar.
  • State 2: Listening. Waveform animation in conversation area. Mic turns terra. Partial transcript fades in.
  • State 3: After first turn. nudgii confirms items as sage chips, asks for more with contextual suggestions.
  • State 4: Continue suggestion. Purple-tinted bubble with "Show my schedule" CTA chip. "or browse instead" link appears below input bar.
This lo-fi shows State 4 (most complete). See hi-fi for all 4 states.
Pattern Copy direction
nudgii's opening frames it as discovery, not inventory. "What do you have at home?" not "What do you own?" The user doesn't need to know what maintenance is required, that's nudgii's job.

Tone: warm, direct, gently funny. Never clinical, never a form. Single question per turn, italic CTA-colored emphasis on the key question within a bubble.

"2-3 items is plenty to start." reduces cold-start hesitation. Users who think they need to add everything at once will freeze.
