Shell + layout
Rule Centered content pattern
Same layout as S-08 Sign-in: vertically centered content zone + sticky bottom CTAs. No scroll needed, content is short. Progress bar at 90%. No back button (forward-only flow, iOS swipe-back gesture handles navigation).
Decision Pre-prompt before iOS dialog
iOS only shows the native permission dialog once. If the user taps "Don't Allow", push is permanently disabled until they manually go to Settings. The pre-prompt gives context and dramatically increases opt-in: ~40% without vs ~70% with a good pre-prompt.
Content
Icon container
56x56px, border-radius 16px, subtle cream bg rgba(26,22,18,0.04), thin border. Bell icon (Phosphor BellSimple, 26px, ink stroke). Matches the SSO screen's icon container pattern for visual consistency across the final onboarding screens.
Headline
DM Serif Display 22px: "Stay on top of things." Single serif moment for this screen. Positioned, not anxious. Doesn't say "allow" or "enable", because nudgii doesn't beg.
Rule 3 benefit rows
Three concrete promises that reduce permission anxiety:

1. Weekly overview every Monday (calendar icon, CTA purple)
2. One nudge on the day itself (bell icon, CTA purple)
3. Silent on other days (mute icon, CTA purple)

"Silent on other days" is the most persuasive line. It answers the user's unspoken fear: "Will this spam me?" Icon containers: 28x28px, CTA purple tinted bg rgba(155,127,212,0.06).
CTAs + states
Primary CTA
"Turn on notifications" (colorCta bg, pill shape, 50px height). Tapping triggers the iOS native permission dialog immediately. Pressed state: bg #7D62B3, scale 0.97, 100ms ease-out.
Ghost CTA
"Maybe later" (transparent bg, colorMidAccessible text, 40px height). Skips to S-08 Sign-in without requesting permission. User can enable later from profile settings. Not hidden or whispered: a visible, guilt-free exit.
Decision iOS native dialog
After "Turn on notifications" tap, iOS shows its native dialog: "nudgii Would Like to Send You Notifications". This text is set in the app's Info.plist (NSUserNotificationUsageDescription). Cannot be customized beyond the app name. Two system buttons: "Don't Allow" / "Allow".

If allowed: brief success (checkmark animation on bell icon, 400ms), then auto-advance to S-08.
If denied: screen dismisses, proceeds to S-08. No error, no retry, no guilt. The user made their choice.
If "Maybe later": skips iOS dialog entirely, proceeds to S-08.
