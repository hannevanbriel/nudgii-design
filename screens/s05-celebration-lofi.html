Screen structure
Rule Shell pattern
Both screens follow the standard shell:
Fixed top: progress bar (75% on S-07a, 100% on S-07)
Expanded: scrollable content zone
Fixed bottom: sticky CTA

No back button on S-07a. The schedule reveal is a one-way moment: the user cannot go back and un-see their schedule. Going forward triggers auth (S-06).
Pattern AHA moment psychology
S-07a is the most critical screen in onboarding. It is where the user sees value for the first time: "I told nudgii what I own, and it built my entire maintenance schedule."

Show value before asking for an account. The schedule is revealed before S-06 (auth). This reverses the typical pattern where apps ask you to sign up before showing what you get.

The value box reinforces the core proposition: you said what, nudgii figured out when.
Note No overdue colors
All dates on S-07a use apricot-dk (the soonest task) or mid (neutral). No terra/overdue colors appear here because the user just created their schedule. Nothing is overdue yet.

This is deliberate: showing overdue items immediately after onboarding would feel punishing instead of empowering.
Note Value box messaging
The value box uses a CTA-purple tinted background to stand out from the nudge cards above it. Copy structure: "You told us X. nudgii figured out Y." This frames the user as the contributor and nudgii as the engine.

In hi-fi, this box gets background: rgba(155,127,212,0.08) with a 1px solid rgba(155,127,212,0.15) border.
Flow and transitions
Interaction S-07a to S-06 transition
"Save my schedule" does not save directly. It navigates to S-06 (auth), where the user creates an account (Apple, Google, or magic link).

The CTA copy ("Save my schedule") reframes sign-up as protecting something valuable the user already built. This converts better than "Create account" because the user has something to lose.
  • S-07a "Save my schedule" leads to S-06
  • S-06 auth completes, leads to S-07 celebration
  • S-07 "Open my schedule" leads to S-09 dashboard
Celebration screen (S-07)
Rule Plum background rules
colorPlum #2D1F4A appears ONLY on the done overlay and this celebration screen. It is never used for chips, headers, badges, or any other surface.

ii mark uses the light-on-dark variant: left pill cream (#F5F0E8), right pill CTA purple (#9B7FD4), dot apricot (#C4A8E8).

Confetti dots use category colors (home, vehicle, garden, subscriptions) at low opacity. No gradients, no particle effects. Just scattered dots.
Pattern Personalized humor
"Your boiler is relieved" adapts to the user's first added item. Examples:
  • Boiler: "Your boiler is relieved."
  • Dishwasher: "Your dishwasher approves."
  • Car: "Your car just exhaled."
  • Olive tree: "Your olive tree sends its regards."
This makes the completion feel personal, not generic. Humor is earned because the user just accomplished something.
Note First nudge pill
The glass-morphic pill anchors the celebration in something concrete: "Your first reminder is already scheduled."

Style: background: rgba(255,255,255,0.06); border: 1px solid rgba(255,255,255,0.1); backdrop-filter: blur(4px). Shows "First reminder" + separator dot + the date of the soonest task. Below it, the task name in very faint text.

This prevents the celebration from feeling hollow. The user knows exactly when nudgii will first reach out.
Interaction CTA: never auto-advance
"Open my schedule" navigates to the dashboard (S-09). The user must tap it.

Never auto-advance from S-07. The celebration is a pause moment. Auto-advancing would undercut the feeling of accomplishment. "Tap when you're ready. No rush." reinforces that the user is in control.
Interaction Spring animation
On S-07 load, the ii mark and headline animate in with a spring curve:
400-450ms cubic-bezier(0.34, 1.56, 0.64, 1)

ii mark: scales from 0 to 1. Headline: translateY from 20px to 0. Confetti dots fade in with staggered delays (50ms apart).

This is the only theatrical motion in the app. Every other animation is functional (250ms ease-in-out or faster).
