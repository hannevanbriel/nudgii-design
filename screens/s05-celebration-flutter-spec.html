Design hub / S-05 Celebration, Flutter spec Dev handoff

1. Screen overview

What this screen does

S-05 is a two-state screen. State 1 shows the user's generated maintenance schedule (the AHA moment). When the user taps "That's it. I'm done.", the screen transitions to State 2: a full-screen celebration on a plum background with confetti, the ii mark, a headline, and a reminder pill. After 3.5 seconds, a sign-in bottom sheet slides up from below.

  • Source prototype: s05-celebration-prototype.html
  • Progress bar fill: 80% in State 1, animates to 85% when celebration starts
  • No back button. Forward-only flow. iOS swipe-back gesture handles navigation.
  • No tab bar. This is an onboarding screen.

2. Animation timeline

All timings are measured from the moment the user taps the CTA button. The orchestration uses a single AnimationController for the celebration sequence (0-900ms), then a delayed Future for the bottom sheet at 3500ms.

  bool _showSheet = false;

  @override

    super.initState();
    _celebrationController = AnimationController(
      vsync: this,
      duration: const Duration(milliseconds: 900),
    );

      vsync: this,
      duration: const Duration(milliseconds: 350),
    );
  }

  @override

    return Scaffold(
      backgroundColor: _isCelebrating ? colorPlum : colorCream,
      body: SafeArea(
        top: false,
        child: Column(
          children: [
            // Progress bar (outside SafeArea, full-width)
            _buildProgressBar(),
            Expanded(
              child: Stack(
                children: [
                  // State 1: schedule reveal

                  // State 2: celebration (always mounted, hidden until triggered)

                  // Scrim

4.6 Confetti system

// 40 confetti pieces. Each has:
// - Random horizontal position (5%-95% of width)
// - Random size (3-9px width, 6-18px height for rects, square for circles)
// - Random color from category palette
// - Random fall distance (500-800px)
// - Random fall duration (2.5-5.0s)
// - Random spin (180-720 deg)
// - Random horizontal sway
// - Staggered start: piece[i] starts at 250ms + i*30ms

// Recommended approach: CustomPainter with a list of ConfettiParticle objects.

class ConfettiParticle {
  final double x;         // 0.05 to 0.95 (fraction of width)
  final double width;
  final double height;
  final Color color;
  final bool isCircle;
  final double fallDistance;
  final double fallDuration; // seconds
  final double spin;         // radians
  final double swayAmount;
  final double swayDuration;
  final double opacity;      // 0.6 to 0.95
  final double delay;        // seconds from confetti start
}

// Colors (all 13 from prototype):
const confettiColors = [
  Color(0xFFC87850),  // home warm
  Color(0xFFD4956A),  // home warm light
  Color(0xFF9B7FD4),  // cta purple
  Color(0xFFB49EE0),  // cta purple light
  Color(0xFF5A9060),  // garden green
  Color(0xFF7AAC7E),  // garden green light
  Color(0xFF4A82A0),  // vehicle blue
  Color(0xFF6BA0B8),  // vehicle blue light
  Color(0xFFE8A87C),  // apricot
  Color(0xFFF0C4A0),  // apricot light
  Color(0xFF8060A8),  // subscription purple
  Color(0xFFA080C0),  // subscription purple light
  Color(0xFFF5F0E8),  // cream
];

// Shape distribution: 60% rectangles, 40% circles (matching prototype ratio)

// Each particle animation:
// - Fall: translateY from -20px to fallDistance, ease-in, duration = fallDuration
// - Appear: opacity 0 to target at 8% of fall duration, then 0 at 100%
// - Sway: margin-left oscillation, ease-in-out, looping
// - Spin: continuous rotation, var degrees

// CustomPainter approach:
class ConfettiPainter extends CustomPainter {
  final List<ConfettiParticle> particles;
  final double progress; // 0.0 to 1.0 from AnimationController

  ConfettiPainter({required this.particles, required this.progress});

  @override
  void paint(Canvas canvas, Size size) {
    for (final p in particles) {
      // Calculate individual progress based on stagger delay
      final particleProgress = ((progress * 5.0) - p.delay).clamp(0.0, 1.0);
      if (particleProgress <= 0) continue;

      // Fade: appear quickly, then fade out
      double opacity;
      if (particleProgress < 0.08) {
        opacity = (particleProgress / 0.08) * p.opacity;
      } else {
        opacity = p.opacity * (1.0 - particleProgress);
      }
      if (opacity <= 0) continue;

      // Position
      final x = p.x * size.width + sin(particleProgress * pi * 2 / p.swayDuration) * p.swayAmount;
      final y = -20 + particleProgress * p.fallDistance;

      canvas.save();
      canvas.translate(x, y);
      canvas.rotate(particleProgress * p.spin);

      final paint = Paint()
        ..color = p.color.withOpacity(opacity)
        ..style = PaintingStyle.fill;

      if (p.isCircle) {
        canvas.drawCircle(Offset.zero, p.width / 2, paint);
      } else {
        canvas.drawRRect(
          RRect.fromRectAndRadius(
            Rect.fromCenter(center: Offset.zero, width: p.width, height: p.height),
            const Radius.circular(1),
          ),
          paint,
        );
      }
      canvas.restore();
    }
  }

  @override
  bool shouldRepaint(ConfettiPainter oldDelegate) => oldDelegate.progress != progress;
}

Use a separate AnimationController for confetti with a 5-second duration (longest piece fall = 5s). Start it at the 250ms mark.

4.7 Bottom sheet (sign-in)

// The bottom sheet appears 3500ms after the celebration starts.
// It slides up from translateY(100%) to translateY(0) over 350ms ease-out.
// A scrim (rgba(0,0,0,0.25)) fades in simultaneously over 250ms.

// Trigger with Future.delayed:
void _startCelebration() {
  setState(() => _isCelebrating = true);

  // Start celebration animations after 150ms
  Future.delayed(const Duration(milliseconds: 150), () {
    _celebrationController.forward();
    _confettiController.forward();
  });

  // Show bottom sheet after 3500ms
  Future.delayed(const Duration(milliseconds: 3500), () {
    if (!mounted) return;
    setState(() => _showSheet = true);
    _sheetController.forward();
  });
}

// Scrim:
AnimatedContainer(
  duration: const Duration(milliseconds: 250),
  curve: Curves.easeOut,
  color: _showSheet ? Colors.black.withOpacity(0.25) : Colors.transparent,
)

// Sheet slide:
SlideTransition(
  position: Tween<Offset>(
    begin: const Offset(0, 1),  // off-screen below
    end: Offset.zero,
  ).animate(CurvedAnimation(
    parent: _sheetController,
    curve: Curves.easeOut,
  )),
  child: _buildSheetContent(),
)

9. Edge cases

User taps during celebration (before sheet appears)
No action. The celebration is non-interactive. Taps on the plum area do nothing. The confetti layer has pointer-events: none. The user waits for the sheet to appear at 3500ms.
User taps "Not now" on sign-in sheet
Sheet slides down (reverse of slide-up, 250ms ease-in), scrim fades out. Navigate to S-06 (push permission). User is treated as anonymous. Dashboard greeting: "Good morning, there." Weekly digest prompt on dashboard will offer SSO sign-in bottom sheet (not email capture).
User taps scrim behind sheet
Same behavior as "Not now". Dismiss sheet, navigate forward.
Successful Apple/Google sign-in
SSO flow completes natively. On success callback: dismiss sheet, navigate to S-06 (push permission). User's name is now available. Dashboard greeting: "Good morning, Martijn." Set up Supabase Auth session.
SSO flow cancelled or failed
Sheet stays visible. If native SSO was cancelled by user, no error shown. If network error, show a toast (error style: red-lt background, red text, "Something went wrong. Try again.") positioned center-bottom, 14px above sheet bottom edge. Toast auto-dismisses after 2.5s.
iOS swipe-back gesture
If user swipes back during State 1 (before tapping CTA), navigate back to S-04 (review screen). If user swipes back during celebration or with sheet visible, block the gesture. The celebration is a forward-only moment.
Screen rotation or multitasking
Lock to portrait. nudgii does not support landscape on any screen.
App backgrounded during celebration
If the app is backgrounded and resumed: if the celebration had not completed, skip directly to the end state (all elements visible, sheet showing). Do not replay the animation. If the sheet was already visible, restore to that state.
