Layout + shell
Rule Shell pattern
SafeArea + Column + Expanded. Nav bar (84px) and FAB are outside scroll area. Content zone: inset: 54px 0 84px 0. Padding: 24px horizontal. Scrollbar hidden. Status bar: 54px total.
Rule Nav bar
Solid var(--surface) background, no glassmorphism. Active tab: plum icon + plum label, no background shape/container. Inactive: var(--hint). Tab order: Today, Plan, My stuff, Profile. Fixed, never scrolls. border-radius: 20px, subtle upward shadow.
Rule FAB
46x46px, border-radius: 14px, plum bg. Plus icon 17px, rotates 45deg to X on open. box-shadow: 0 6px 20px rgba(155,127,212,0.45). Position: right: 24px; bottom: 18px, z-index 45, margin-bottom: 4px.
Greeting
Greeting Variants
Noto Serif 24px, line-height: 1.12, letter-spacing: -0.02em. Two lines: "Good morning," then name on second line. Greeting word varies by time of day: "Good morning," / "Good afternoon," / "Good evening,". Name in apricot italic. Signed-in: first name from SSO profile. Skipped auth: "there." ii mark avatar (44x44, surface-low circle) sits left. Date line: Manrope 12px, var(--mid), left-aligned under greeting with 58px left padding.
Summary line
Summary Inline text + streak
No card, no progress ring. Flex row, items centered. Left: summary text (13px 500 ink, flex 1), e.g. "3 things need attention today". Right: streak circle (46px, apricot-wash bg, apricot number 17px 600, "week/streak" 6px). Streak is tappable: tooltip auto-dismisses after 2.5s. When all overdue: summary text uses apricot color instead of ink.
Task cards
Task card Collapsed
40px round icon circle with category wash bg + category-colored icon SVG (18px). No card background wash on the row itself. Title: 13px 500 ink. Subtitle: 10px mid (item name + frequency). Right side: date badge ("Wed 26" in mid) or overdue text ("12d overdue" in apricot 600). Chevron: 12px, 0.35 opacity, rotates 180deg on expand.
Task card Expanded
Chevron rotates. "Why it matters" block appears: surface-lowest bg, 8px radius, apricot label (9px 600), body text (11px mid). Below: action row, space-between layout. Left: "Details" link (10px 500 mid) with chevron. Right: Skip (hint, no bg) + Snooze (ink text, surface-low pill) + Done (category color pill with checkmark). max-height transition 300ms ease.
Decision Done button color
Done button uses the category color as its fill (e.g. var(--ch-bg) for Home tasks). Not plum. Plum is reserved for app-level actions only (FAB, CTA, active tab). Category color on the Done button reinforces which category the task belongs to and keeps plum meaningful. Skip = no bg, hint text. Snooze = surface-low pill.
Decision Done: moves to "Done today" section
Tapping Done does not remove the task from the list. Instead, it animates down (spring, 350ms) to a "Done today" section at the bottom of the Today panel. This gives users a same-day memory log: "Did I already do that?" is answerable without opening history.

Done today row: checkmark circle (category color) + muted title (55% opacity) + time or "just now". Section label: "Done today" (9px uppercase hint). Resets at midnight — items move to S-16 completion history.

When all tasks are done: Done today section is the only content. Summary line updates to "All done for today." Witty line appears below the Done today list.

Undo toast (5s) still available: task springs back to its original position in the list and is removed from Done today.

Tapping "Done today" section label or any done item navigates to S-16 (completion history). Flutter: animate with AnimatedList, remove from source list, insert into done list. Use task.completed_at timestamp for "just now" logic.
Decision Skip: this occurrence only
Skip = this occurrence only. The recurring schedule is never affected. The task returns on its next due date.

Data model: task.last_completed_at is never updated by skip. task.last_skipped_at is set. task.next_due_at recalculates as last_skipped_at + interval.

Inline preview before collapse (not a toast): When Skip is tapped, a brief line appears inside the expanded card for 800ms: "Skipping this time. Back on May 24." Then card collapses (spring out). Feedback is in context, not at screen bottom.

Copy variants: recurring with date = "Skipping this time. Back on [date]." Recurring no fixed date = "Skipping this time. Next one in 6 weeks." Non-recurring = "Skipped. This task won't repeat."

On 2nd skip of same task within 30 days: "Skipping again. Still counting from March 1." (honest, non-naggy). On 3rd skip in a row: For You section surfaces a gentle suggestion: "You've skipped this 3 times. Want to reschedule it?"

Flutter: task.skipCount field, increment on skip. Check against threshold in For You card algorithm.
Decision Snooze: half-height bottom sheet
Tap Snooze: half-height bottom sheet slides up (280px, surface bg, 20px top radius, drag handle). 4 preset options:

"Later today" (3h from now), "Tomorrow" (8:00 AM), "This weekend" (Sat 8:00), "Next week" (Mon 8:00).

Plus secondary row: "Pick a date..." with chevron (opens date picker). Tap any preset: sheet dismisses instantly, task disappears from list, toast "Snoozed until Saturday."

Escalating copy on repeat snooze: 1st = neutral, 2nd = "Fine. But your dishwasher is taking notes.", 3rd = "Your boiler forgives you. For now." Flutter: task.snoozeCount field, check in copy resolver.
For you (discovery cards)
Spec Full card system documented
Card types, priority algorithm, Free/Pro/Ultimate matrix, rotation + dismissal rules are all defined in a dedicated spec page.

Open For You: Card System →
Cards Horizontal scroll row
Horizontally scrollable, 12px gap. Cards: min 175px, max 200px, surface-low or category wash bg, 12px radius, 16px padding. Dismiss X: 22px circle, ink 5% bg. Types: "Did you know?" (category wash, CTA button), "Saved this year" (surface-low, Noto Serif 26px apricot number), "Spring ready" (garden wash, progress dots), seasonal tips. Free users cannot hide the row. Individual cards dismissible, new ones rotate in.
Section: THIS WEEK
Decision Calendar week boundary
THIS WEEK shows tasks from tomorrow until Sunday (ISO calendar week, Mon–Sun — standard in NL/BE). Not a rolling 7-day window.

Why: home maintenance tasks cluster on weekends. "This week" maps to how users actually plan ("I'll do that this weekend").

Sunday exception: on Sunday, THIS WEEK would be empty (week ends today). Instead, on Sundays the section looks ahead to the full coming week (Mon–Sun of next week). Label descriptor changes to "coming week · X tasks". Sunday becomes a natural planning day. Section never goes empty because of the day of the week.

Flutter: isSunday branch in section builder. Mon–Sat: query dueDate > today AND dueDate <= thisSunday. Sunday: query dueDate >= nextMonday AND dueDate <= nextSunday. Section is always rendered — never hidden. If query returns empty, show italic hint text "All clear until Sunday." (or "All clear next week." on Sundays). Use tasks.isEmpty ? EmptyWeekHint() : TaskList(tasks).
Section: COMING UP
Section Collapsible
Label: "coming up" + " · next 30 days · X tasks" (10px mid). Chevron right, rotates 180deg on expand. Tasks inside use .feat style (same as TODAY tasks) but without expand — tap goes to task detail. Collapsed by default. Section visible in Default state and Overdue only state (future tasks are not affected by today's overdue tasks).
States — when each one fires
State 1 Default
When: mix of tasks — some overdue, some due today, some later this week or coming up.

Summary: "X things need attention today" (ink). Streak tappable. Sections: TODAY (overdue + due today merged) → THIS WEEK → COMING UP (collapsible) → FOR YOU.
State 2 Quiet day
When: user has items and tasks, but nothing is due today or overdue. Next task is later this week or further out.

Summary: "Nothing due today." (ink — this is a good state, no apricot). Streak intact. TODAY section shows empty hint (rotates). THIS WEEK shows upcoming tasks normally. COMING UP collapsible. FOR YOU visible.
State 3 Overdue only
When: every task due today or earlier is still pending. Nothing due later this week.

Summary turns apricot: "X things could use you". Streak dimmed at 0 (broken). TODAY shows only overdue tasks. THIS WEEK always visible — empty hint + "Review your items →" action. COMING UP visible collapsed. FOR YOU visible. Witty closer: "Not judging. Much."
State 4 All done
When: all tasks due today and earlier marked Done or Skipped. Nothing pending.

Summary: "Nothing due. Suspicious." Streak incremented. Celebration block: apricot-wash circle + checkmark + "All caught up" + next task preview. FOR YOU visible — reward for completion. Fires after plum done overlay dismisses.
State 5 No items
When: user has not added any items yet (brand new account, or all items deleted).

Greeting: "Good morning, there." (no name). No summary, no task sections. Centered: calendar icon + "Nothing here yet" + body copy + plum pill CTA "Add your first item". No FOR YOU (nothing to base suggestions on).
Empty state copy — rotation lists
Copy TODAY empty (quiet day)
Shown when nothing is due or overdue today. Rotates randomly on each app open.

1. "Nothing due today. Enjoy it."
2. "Quiet day. Make the most of it."
3. "Nothing today. Your house is behaving."
4. "All clear today."

No CTA when TODAY alone is empty — user is on top of things. CTA only appears when TODAY + THIS WEEK are both empty (see "too empty" rule below).
Copy THIS WEEK empty
Shown when nothing is due between tomorrow and Sunday (or next week on Sundays). Always includes a soft action link.

1. "Coast is clear until Sunday."
2. "All quiet until Sunday."
3. "Nothing on the radar until Sunday."
4. "Your weekend is yours." (Mon–Thu only, not Fri–Sat)

No CTA. An empty THIS WEEK is not a problem — nothing is wrong, nothing needs fixing. A link would imply otherwise. Italic hint text only.
Copy COMING UP empty
Shown when nothing is due in the next 30 days. Strongest signal that user needs more items.

1. "Nothing in the next 30 days." + sub: "Add more items and nudgii will fill this in."
2. "Clear horizon." + sub: "You might be missing some items."
3. "Your future self has no complaints." + sub: "Though nudgii might have some suggestions."

Action: + Add items (plum text link, opens FAB or My stuff). More prominent than THIS WEEK action — this is a real gap signal.
Rule Too empty — all three empty
When: TODAY + THIS WEEK + COMING UP are all empty simultaneously. This means the user has very few items or tasks aren't set up correctly.

Action: FOR YOU section gets a persistent fallback card that cannot be dismissed (no X button). Card: surface-low bg, nudgii icon, "Your schedule is looking quiet." body copy + "Add more items and nudgii will figure out the rest." + + Add items plum pill CTA.

Flutter: bool isAllEmpty = todayTasks.isEmpty && weekTasks.isEmpty && comingUpTasks.isEmpty. If true, inject persistent card as first item in FOR YOU list. isDismissable: false.
Dark mode
Rule Dark tokens
Surface: #1C1B18 (warm dark). Plum lightens to #B8A0E8, apricot to #F0B890. Category washes at 15% opacity (was 14% light). Text primary: #F0EDE6. Mid: #A8A29A. Nav bar: surface-low (#24231F). No blue or cool-green tints. Toggle with [data-theme="dark"].
Typography
Rule Hierarchy
Greeting: Noto Serif 28px. Summary text: Manrope 13px 500. Section labels: Manrope 10px 600, uppercase, letter-spacing: 0.05rem. Task title: 13px 500. Task subtitle: 10px. Badges: 9px. Witty closer: Noto Serif italic 13px, hint color, 0.55 opacity.
Responsive notes
Long titles
Task titles: min-width: 0 on flex child, natural text wrap at 2 lines max. Badge and chevron never shrink (flex-shrink: 0). Icon circle never shrinks. On very long titles, the second line wraps naturally. No ellipsis truncation.
