Tile grid browse
A+BStatus bar + progress bar
Standard iOS notch + status bar. Progress bar: full-width, edge-to-edge, 3px, colorCta #9B7FD4 at 40% (S-03 = step 2). Outside SafeArea. Fills 8% per selected item, clamped at 80%. Animates on every toggle.
CBack button
PhosphorIcons ArrowLeft: horizontal line + arrowhead. 44x44px tap target. Returns to S-01.
DHeader
DM Serif Display 20px. "What's" plain, "yours?" italic in colorCta. Subtitle: DM Sans 11px italic colorMid. One serif moment per screen.
ESearch bar
White surface, 0.5px border, 10px radius. MagnifyingGlass icon 13px. Live filter: filters visible tiles as user types. Shows flat grid when query present. Clears when category chip is tapped.
FCategory chips — new style
No dots. No borders. Each chip uses category-tinted background + category color text.
Home chip: rgba(200,120,80,0.13) bg, #C87850 text.
Vehicle chip: rgba(74,130,160,0.13) bg, #4A82A0 text.
Garden chip: rgba(90,144,96,0.13) bg, #5A9060 text.
Subscriptions chip: rgba(128,96,168,0.13) bg, #8060A8 text.
Active category: adds 1px solid categoryColor outline.
"All" inactive: rgba(26,22,18,0.07) bg, colorMid text, no outline.
"All" active: rgba(155,127,212,0.12) bg, 1px solid colorCta outline, colorCta text.
Single scrollable row. No GestureDetector dot, no FilterChip widget. Use GestureDetector + AnimatedContainer for smooth transitions.
GSelected pill
Appears when selectedIds.isNotEmpty. colorCta filled pill, white text, count in a small white-tinted circle left side.
Tapping it shows only selected items in a 3-col grid. Acts as a quick filter for review.
HTile rows — overview mode
One section per category in order: Home, Vehicle, Garden, Subscriptions. Section header: 9px, 600 weight, 1.3px letter-spacing, uppercase, colorMidAccessible + 6px colored dot.
Each section: horizontal SingleChildScrollView row of tiles. Padding: EdgeInsets.symmetric(vertical:4, horizontal:16). Critical: vertical padding prevents selected border from clipping.
Swipe hint (first load only): Home row scrolls 52px right, pauses 520ms, snaps back. Teaches the swipe pattern. hintDone flag prevents repeat.
End of each row: faint › peek indicator (shows more tiles exist). rgba(26,22,18,0.20).
Single category mode (chip tapped): Switches to 3-column GridView of all items in that category. Selected items sort to the top.
HTile widget
88px wide, radius 14px. Background: category color at 12% opacity. Border: 1.5px solid transparent (unselected) / 1.5px solid categoryColor (selected, NOT purple).
Icon: Phosphor outline, 26px, category color. No inner wrapper. Rendered directly on tile background.
Label: DM Sans 9.5px, weight 500, colorInk, centered, max 2 lines.
Hover (web): bg lifts to category color at 20%, scale 1.03.
Press: scale 0.93. Haptic: HapticFeedback.selectionClick() on every toggle.
ISticky CTA
Full width, 50px, radius 100px. Above: 0.5px hairline separator.
0 items: rgba(26,22,18,0.08) bg, colorMid text "Tick something first". Not tappable.
1 item: Active CTA, "Good start, continue".
2 items: "Nice, keep going or continue".
3+ items: "Looking good, continue".
Copy transition: fade out 180ms, swap, fade in. Count badge in white-tinted circle, right side.
Rules
Locked decisions
Tiles, not rows
Visual tile grid replaces flat list rows. More items visible above the fold. Faster for Arne: tap to recognize, not read.
No dots on chips
Category chips use tinted bg + category color text. No dot indicator. Cleaner. Color conveys category identity.
Category color on tile border (selected)
Selected tile border = category color, not purple. Purple is reserved for CTA/active filter. Category owns its visual.
Swipe hint once only
Home row scroll hint fires on first load only. hintDone flag. Never repeated — doesn't teach twice what users learn once.
Progress bar grows with selections
40% base + 8% per item, clamped at 80%. Reinforces that adding items = moving forward.
CTA copy progression
"Tick something first" → "Good start" → "Nice, keep going" → "Looking good, continue". Friendly, progressive, never pushy.
Navigation
Adjacent screens
Comes from: S-01 Welcome, Path B CTA "Show me what I can track".
Context carry-over: No items pre-selected (S-01 no longer has interactive selection as of v2). S-03 always opens fresh.
Goes to: S-04 Review, via CTA (any active state).
Back button: Returns to S-01.
