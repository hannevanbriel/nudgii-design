Flat browse with filters
A+BStatus bar + progress bar
Standard iOS notch (24px) + status bar. Progress bar: full-width, edge-to-edge, 3px, colorCta #9B7FD4 fill at 40% (S-03 = step 2 of 5). Outside SafeArea. Never segmented, never dots.
CBack button
PhosphorIcons ArrowLeft: horizontal line + arrowhead. SVG paths d="M10 6H2" + d="M5 3L2 6L5 9" in viewBox 0 0 12 12. 44x44px tap target. Navigates back to S-01.
DSearch bar
Magnifying glass icon + "Search items..." placeholder. White surface bg, border-radius 10px, 0.5px border. Live filter: filters visible items as user types. No separate search screen.
ECategory filter strip
"All" pill always first, always active initially. Single scrollable row (flex-wrap:nowrap; overflow-x:auto). Category pills include a 6px colored dot + label.
Active pill: background:var(--ink); color:#F5F0E8; border-color:var(--ink)
Inactive pill: surface bg, 0.5px border, colorMidAccessible text.
Tapping a category pill filters the item list below to only show that category.
FItem list (grouped by category)
Items grouped under section headers: 9px, 500 weight, 1.8px letter-spacing, uppercase, colorMidAccessible.
Each item = individual white card with 6px gap. No border-bottom separators. Card: 0.5px border, 10px radius, subtle shadow.
Each row: category icon chip (22x22px) + item name (10.5px/500) + reminder count pill ("3 reminders") + check circle (right side, 18px diameter).
Pre-checked items: Items selected on S-01 arrive pre-checked (Dishwasher, Car in this example). Selected state: 1.5px CTA border + subtle CTA tint bg.
"Something else" row: Dashed border, plus icon. Discovery path for items not in the default list.
GSticky CTA
"Continue with X items" label updates dynamically as items are checked/unchecked. Count badge shows the number. Navigates to S-05 (review).
Disabled state when 0 items selected: rgba(155,127,212,0.35) bg, rgba(250,248,255,0.5) text.
Rules
Locked decisions
Flat list, not category tiles
Single scrollable list with section headers replaces the 2x2 category grid. Faster for Arne: fewer taps to select items.
Category colors only in chips
catHome/catVehicle/catGarden/catSub appear only in category icon chips. Never on status, tab bar, or buttons.
Individual white cards
Items are always individual cards with gap between them. Never border-bottom separators. Cream gap IS the separator.
"All" pill present
"All" pill on S-03 is a functional filter (unlike S-01 where it was removed as decorative). Always first, always active initially.
Navigation
Adjacent screens
Comes from: S-01 Welcome, Path B CTA "Show me what I can track".
Context carry-over: Items pre-selected on S-01 arrive checked. Never re-ask for information already given.
Goes to: S-05 Onboarding review, via CTA "Continue with X items".
Back button: Returns to S-01. No intermediate states.
