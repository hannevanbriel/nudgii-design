Shell + layout
Rule Progress bar
Full-width, edge-to-edge, outside SafeArea. 3px height, colorCta #9B7FD4 fill at 60% (S-05 = step 3 of 5). Track: rgba(26,22,18,0.08). Never segmented, never dots. Flutter: LinearProgressIndicator inside ClipRRect(borderRadius:100px).
Rule Back button + scroll zones
Back button: PhosphorIcons.arrowLeft, 44x44px tap target. SVG: horizontal line + arrowhead, viewBox 0 0 12 12. Content zone scrolls (headline through add-another link). CTA button is fixed at bottom, outside scroll area. Flutter: Column with Expanded(child: SingleChildScrollView) + fixed bottom Column.
Content
Note Headline + subtitle
Headline: "Here's what nudgii found." in 16px DM Sans 500, colorInk. Subtitle: "Scan to confirm brand or model. You can adjust anything later." in 11px italic DM Sans, colorMidAccessible. Reduces review friction: reminds user the AI scan is available, "adjust later" lowers perfectionism.
Decision Review cards expand/collapse
Tap card header to toggle expanded/collapsed. First card expanded by default, remaining cards collapsed. Expanded state: tasks list + "last done" picker + scan button visible. Collapsed state: task count hint ("3 tasks · tap to review") + chevron + remove button. Flutter: AnimatedContainer (250ms ease-in-out) for height transitions.
Decision "Last done" picker
3-tap single-select: Recently (delays first task 6-8 weeks), A while ago (schedules sooner), Not sure (default, conservative schedule, tasks appear soon). "Not sure" is always the active default. Flutter: ChoiceChip with single selection. Changes task scheduling logic on the backend.
Decision Scan button placement
Scan button appears in each expanded card header (20x20px rounded square with viewfinder icon). Triggers camera for AI brand/model detection via Claude API. Returns item_type, brand, model, tasks[]. Hidden when card is collapsed.
Decision Remove button
X icon per card header, visible in both expanded and collapsed states. Removes item from review list. Flutter: confirm dialog ("Remove [item name]?") before removeItem(). No undo toast on this screen (items can be re-added via "+ Add another item").
Upsell
Decision Pro lock pattern
3rd task row in each card is greyed out (opacity 0.4) with a "Pro" pill (colorCta text, purple-tinted background). Visible during setup, does not block anything. Soft upsell: user sees what they'd get with Pro without being stopped. Flutter: GestureDetector on Pro pill navigates to S-22 paywall.
Note Item counter
When approaching the 10-item free tier limit, a subtle banner appears above the card list. Not shown in this mockup (2 items). Copy at limit: "You've hit 10 items. Your house has more opinions." Links to S-22 paywall.
Button zone
Rule "Looks good" CTA
Proceeds to S-06 auth. Same button specs as S-01 Path A: colorCta fill, 50px height, pill radius, box-shadow: 0 2px 12px rgba(155,127,212,0.32). Arrow entity in button text. Sentence case. Sticky at bottom, never scrolls.
Changelog
v1.0 · 2026-03-24 · ADDITIVE
Initial hi-fi based on onboarding-full-preview.html (source of truth).
Review cards: expanded state shows tasks + "last done" picker + scan button. Collapsed state: task count hint + chevron + remove button.
"Last done" 3-tap picker: Recently / A while ago / Not sure (default). Single selection per card.
Pro lock pattern: 3rd task greyed with Pro pill (soft upsell, does not block).
"+ Add another item" link below cards for adding items missed during voice/browse/search.
Standard Flutter implementation block · S-05
Mockup base:    390px width (iPhone 15)
SafeArea:       top=false, progress bar outside SafeArea
Scroll:         SingleChildScrollView for content zone only
Bottom zone:    Sticky: fixed Column at bottom, not Spacer
Progress bar:   Full width, height 3px, outside SafeArea, 60% fill
Scaling:        All visual elements x (screenWidth/390).clamp(0.9, 1.15)
Special:        Review cards use AnimatedContainer for expand/collapse (250ms ease-in-out). "Last done" pills are ChoiceChips with single selection.
