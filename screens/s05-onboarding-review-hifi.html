The dashboard shows their actual items from onboarding — not empty. The user is immediately in the app.
Flow summary: Welcome → [Path A voice / Path B browse / Path C search] → Review → Sign in → Completion → Dashboard. Auth is deferred to just before the database write — after the user has seen their full item list and has a reason to create an account. The completion screen auto-advances after 1.2 seconds; no tap required. The dashboard opens with real data from the session.
MVP voice clarification: nudgii does not speak. Path A shows a text chat bubble ("Vertel me wat je hebt…") that the user reads. The user speaks and the transcript appears live. Claude processes and responds in text. This is one-way voice input + text output — not voice-to-voice.
