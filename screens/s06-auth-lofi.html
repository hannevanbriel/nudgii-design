Zones
Screen layout
AStatus bar
Standaard iOS status bar. Hoogte 20px. Transparant over cream achtergrond.
BProgress indicator — 80% (step 4 of 5)
Continuous progress bar, 3px, full-width, edge-to-edge. Fill: colorCta #9B7FD4. Background: rgba(26,22,18,0.08).
S-06 = 80% filled. Zeigarnik effect: almost-there feeling reinforced. User has completed S-05 — progress is visibly earned.
Cii-mark + headline + subtitle
ii-mark centered (left=ink, right=cta 45% opacity, dot=apricot-dk).
"Almost done." — DM Serif Display 18px. Deferred auth = user has already invested 60s + seen their tasks.
No "Create an account" as opener — that sounds like work.
Subtitle: "Create your free account to save your schedule." — DM Sans 10px, colorMid, centered.
DApple sign-in — always first
Apple always first on iOS — iOS HIG requirement. Do not deviate.
Style: 44px height, white background, grey border rgba(26,22,18,0.18), border-radius 100px.
Label: "Continue with Apple" — sentence case.
EGoogle sign-in — second option
Google as second option. Same style as Apple button. Google logo left of label.
F / G / HEmail path — magic link
Magic link = no password — lowest friction for the email path.
Zone G: email input, 44px, border-radius 10px, placeholder "email address".
Zone H: CTA "Send magic link" — 50px, colorCta background, pill-radius, colorCtaFg text.
On tap: inline "Check your email — link sent." within Zone H. No new screen.
Reassurance line: "Free account · no credit card required" shown below Zone H — 9px DM Sans, colorMid, centered. Reduces hesitation at the account creation moment.
Edge case — auth fails: retry button appears, state NEVER lost. Session stored locally until auth succeeds.
H2Auth failure state
Shown only on failed auth. Inline error card below the CTA — no new screen.
Background: red-lt #FCEBEB · Border: 1px solid rgba(163,45,45,0.18) · all sides — no border-left accent.
Icon-chip: 20×20px, red-lt background, terra icon.
Copy: "Sign in failed. Try again — your data is saved."
Session state preserved — items NOT lost on auth failure. Onboarding state stored locally until auth succeeds.
IPrivacy note
7px DM Sans, colorMid-lt, centered. "By continuing you agree to our Terms of service". Link underline on "Terms of service".
Regels
Locked decisions
Deferred auth
User sees value first (S-01 → S-05), auth comes after review. Never as the first screen.
Apple always first
iOS HIG requirement. Apple login always above Google on iOS. Do not deviate.
Magic link = no password
Lowest friction for the email path. No password to create, no password to remember.
State never lost
Auth failure = retry button, not start over. Onboarding state stored locally until auth succeeds.
Reassurance line
"Free account · no credit card required" shown below magic link CTA. Reduces hesitation at account creation moment.
