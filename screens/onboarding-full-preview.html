Design decisions · open questions · changes from previous preview
S-01 · subtitle options
↑
On screen: "Add what you own. We handle the rest." — simplest, most direct. Action-oriented.
on screen
"Add what you own. We handle the rest." — simplest, most direct
alt A
"You know what you have. nudgii knows when." — splits the value prop perfectly
alt B
(remove it entirely) — headline + task rows communicate everything
S-01 · pre-selection + CV ketel
↑
All rows start unselected. UX best practice — pre-ticking choices feels presumptuous. The check circle shape is self-explanatory. Micro-copy "For example — tap what applies to you:" is enough affordance.
★
Real app adds subtle one-time pulse animation on the check circles at first load — draws the eye without forcing a choice.
↑
CV ketel replaces hedge trim. 95%+ of BE/NL homes have central heating. Annual service is safety-critical and universally forgotten — exactly the nudgii value prop in one row.
S-02 · no artificial item cap
↑
Removed "1 of 3" counter pill. Free tier is 10 items — plenty. An artificial 3-item onboarding cap was a false constraint that created urgency without benefit.
★
10 items × 3 tasks = 30 reminders/year spread across seasons. That's minimal and genuinely useful at no cost. Upgrade trigger is features (AI, sharing, export) — not item count.
↓
AI cost note: Claude API calls per onboarding are naturally bounded by how many items a user adds (typically 2–4). No hard cap needed in the UI — cost stays manageable.
S-05 · "when last done"
N
Replaces status dots entirely. We can't know overdue at onboarding — there's no history yet. Instead, 3-tap quick picker per item: Recently · A while ago · Not sure.
★
Default: "Not sure" → nudgii schedules the first reminder soon (conservative). "Recently" → adds 6–8 wks before first nudge. Personalises the schedule without asking invasive questions.
↑
Remove button (×) per card. Users can remove items from review. "+ Add another item" link at bottom — no slot counter.
↓
Pro lock row per card — 3rd task shown greyed with "Pro" pill. Soft upsell visible during setup without blocking anything.
S-07a · no status colours
↑
Removed terra red "overdue" from AHA screen. We don't know overdue at this point — "last done" was just collected on S-05. All cards show upcoming dates only: apricot-dk for soon, mid for further out.
↑
Changed "Your first week" → "Your first month" — with CV ketel service in Oct and dishwasher tasks in Nov, "first week" was misleading. "First month" is more accurate and still impactful.
S-07 · celebration redesign
↑
Replaced ink background with cream + celebration. Ink felt cold and harsh. Cream stays warm and on-brand — the "arrival" feels earned, not dramatic.
N
Dotted CTA-colour ring around large ii mark. Pure CSS, no gradients. Visual reward without fanfare. Double ring (inner dashed + outer faint) adds subtle depth.
N
Confetti dots — small CSS circles scattered in category colours (home/vehicle/garden/subscriptions). Solid fills only, no gradients, varied opacity. Stays within design rules.
★
Copy: "Your boiler is relieved. We've got it from here." — earned humour, personalised, warm. The boiler reference ties back to the item they just added.
↓
CTA: "Open my schedule →" — required tap, never auto-advance. Note below: "Tap when you're ready — no rush."
S-09 · dashboard · free tier state
N
Upgrade nudge at bottom — "2 of 10 items used. Add up to 8 more. Go Pro for unlimited." Visible but not intrusive. Appears below all tasks, never above.
★
No "overdue" section on first load — user just set up, all tasks are upcoming. Overdue section appears only when something is actually past due.
↓
Tab bar: Tasks (active, purple pip) · Ask · Profile. FAB: colorCta rounded square adjacent to pill. Custom widget — never BottomNavigationBar.
