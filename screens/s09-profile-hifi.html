Nav bar + shell
Rule No FAB on Profile
The FAB (add item/task) is not shown on the Profile tab. It is only relevant on Today, Plan, and My Stuff. Removing it reduces cognitive noise on a settings-focused screen. Flutter: FAB is conditionally rendered based on currentTab != TabIndex.profile.
Rule Nav bar: Profile active
Profile icon (user circle) and label "Profile" both use var(--primary) (plum) for active state. No background shape behind active tab. Inactive tabs: var(--hint) stroke. Tab order is fixed: Today / Plan / My stuff / Profile. Nav bar: solid var(--surface) bg, no glassmorphism. Subtle upward shadow.
User header
Avatar Initials circle
56px circle, var(--apricot-wash) bg, 2px apricot border at 25% opacity. Single initial letter: DM Serif Display italic, 22px, var(--apricot). Name from SSO profile (first name only). Flutter: CircleAvatar with backgroundColor + Text child using GoogleFonts.notoSerif(fontStyle: FontStyle.italic). If user has a profile photo (future), swap initial for NetworkImage.
Plan badge Free vs Pro
Free: var(--sage-lt) bg + var(--sage) text. Pill shape, 10px 500.

Pro: var(--plum-wash) bg + var(--plum) text. Same dimensions.

Flutter: badge color driven by UserPlan.isPro. Use Container with BoxDecoration + BorderRadius.circular(100). No border needed.
Stats circles
Stats Three equal circles
3 equal-flex tiles in a Row, gap 8px. Each: surface-low bg, 50px border-radius (pill-shaped horizontally), min-height 64px, centered content. Number: DM Serif Display 20px, var(--apricot). Label: 8px 500 uppercase, var(--hint), letter-spacing 0.07em. Stats shown: items owned, tasks completed all time, current streak in weeks.

Flutter: Row + 3x Expanded children. Number from Supabase user stats query.
Copy Personality line (rotates)
DM Serif Display italic 11px, var(--hint), 0.65 opacity. Rotates randomly on each app open. Examples:

1. "Your boiler has had 6 check-ups."
2. "You've completed 43 tasks. Not bad."
3. "Your house is in good hands."
4. "4 weeks in a row. Impressive."

Store rotation pool in database. Select by hash of user ID + current week number for consistent display within the same week.
Upgrade to Pro banner
Placement Below stats, above notifications
Banner is placed after stats and personality line, before the first utility section. Rationale: stats create context (user sees what they have achieved), then the upgrade banner feels like a natural next step rather than a gate. Not the very first thing on screen (that would feel pushy). Visible on first scroll without needing to scroll far.

Free state only. Banner is removed entirely on Pro state. No "downgrade" or "manage" equivalent in this position.
Banner Style
surface-low bg, 1px solid rgba(155,127,212,0.20) border, 12px radius, 14px padding. Three-column flex: icon left (32px, 8px radius, plum-wash bg + ii mark) + copy center (title 13px 500 ink, sub 11px mid) + "See Pro" pill right (10px 500 plum-fg on plum bg). Flutter: InkWell wrapping a Container. Taps entire banner area.
Notification preferences
Weekly digest Row
Label left + day/time value right ("Monday 8:00", 11px mid) + pencil icon (12px, 0.5 opacity). Tap opens a bottom sheet with day picker (Mon–Sun) + time picker. Flutter: showModalBottomSheet with ListWheelScrollView or custom picker. Value is stored per-user in Supabase (default: Monday 8:00 in user's local timezone).
Toggles iOS-style SVG
Push reminders: ON by default (plum bg, circle at cx=28).
Dark mode: OFF by default (surface-high bg, circle at cx=12).
Smart nudges: Pro only row, ON by default for Pro users.

SVG toggle spec: width="40" height="24" rx="12". ON: fill var(--plum) + white circle cx=28. OFF: fill var(--surface-high) + white circle cx=12. Flutter: use Switch widget with custom activeColor: colorCta + activeTrackColor: colorCta.withOpacity(0.5). Or custom AnimatedContainer for full fidelity.
Account section
List rows Pattern
Each row: 44px min-height (tap target rule), flex row, 14px padding. Label left (13px, ink). Optional value right (11px, mid). Chevron far right (10px hint, var(--hint)) where row navigates. 1px var(--surface-high) divider between rows inside the same card — never between cards (gap pattern).

Flutter: ListView children inside a ClipRRect container. Dividers are Divider(height: 1, color: colorSurfaceHigh) between items, not on the card itself.
Decision Upgrade CTA in account section
"Upgrade to Pro" row in the Account section (Free only) uses plum text + right arrow, no chevron. This is the second upgrade touchpoint on the screen (after the banner). Placement in "Account" makes it feel like a natural account action, not advertising. The banner is the primary CTA. This row is the secondary, for users who scroll past the banner or dismiss it (future behavior).

Pro state: "Upgrade to Pro" row is replaced by "Manage subscription" (mid text + chevron) and "Household sharing" (mid text + value + chevron).
Settings section
Export my data Plum action
"Export my data" label uses var(--plum) text and a download icon (also plum). This is an app-level action (exporting data), so plum is correct here. Chevron also plum. Tap triggers: confirmation sheet ("Export all your data as CSV?") + export action via Supabase function.
Dark mode toggle Live preview
Toggling dark mode applies the theme immediately (no confirmation, no delay). Flutter: update ThemeMode in Provider or Riverpod state. Persist to Supabase user preferences. The toggle in this UI is visually OFF by default (light mode active). When user taps, circle animates from left to right, bg transitions from surface-high to plum.
Support section
Version row No chevron, no tap
"Version 1.0.0 (42)" row: label only (mid, 12px), no chevron, no InkWell. Value is app version string from package_info_plus plugin. cursor: default in HTML. Easter egg: tap 7x opens debug info sheet (standard mobile pattern). Not shown in hi-fi.
Sign out
Sign out Red text, centered
13px, var(--red), centered, 20px top padding. No button shape, no icon. Tap opens a confirmation bottom sheet: "Sign out?" + "You'll need to sign back in to access your data." + "Sign out" (red pill) + "Cancel" (ghost). No confirmation needed at this tap — confirmation is in the sheet. Flutter: TextButton with style: TextButton.styleFrom(foregroundColor: colorRed).
Free vs Pro differences
Visual diff Free vs Pro
Free: Plan badge = sage green. Stats show lower numbers (fewer items). Upgrade banner visible between stats and notifications. Account section includes "Upgrade to Pro" plum row. No "Household sharing" row. No "Smart nudges" toggle.

Pro: Plan badge = plum. Stats reflect full usage (14 items vs 7). No upgrade banner. Account section: "Plan · Pro · €4.99/mo" + "Manage subscription" + "Household sharing (1 of 3)". Notifications card gains "Smart nudges" toggle (ON). Everything else is identical.
Dark mode
Tokens Dark mode values
Surface: #1C1B18. Surface-low: #24231F. Surface-high: #2E2D28. Plum lightens to #B8A0E8. Apricot to #F0B890. Text primary: #F0EDE6. Mid: #A8A29A. Nav bar: surface-low. No blue or cool-green tints. Plum-wash updates to rgba(184,160,232,0.12). Sage and red semantic colors lighten for dark bg contrast.
Flutter implementation
Flutter Screen structure
Shell: SafeArea + Column. Content: Expanded wrapping ListView (the only scrolling element). Bottom: fixed nav bar (84px).

Sections: use ListView with custom section widgets. Not SingleChildScrollView + Column. Stat circles: Row + 3x Expanded + custom Container. Toggles: Switch widget with custom colors or full custom painter. Avatar: CircleAvatar. Sign out: TextButton.

Data: all profile data from Supabase. Plan state from RevenueCat CustomerInfo. Version from package_info_plus. Stats from aggregate query on tasks + items tables.

v1.0 · 2026-04-02 · ADDITIVE
Initial hi-fi for Profile tab. New file. Two plan states: Free and Pro.

User header: 56px avatar circle with DM Serif Display italic apricot initial. Name (DM Serif Display 24px). "Member since" date (10px mid). Plan badge: sage-lt/sage for Free, plum-wash/plum for Pro.

Stats row: 3 equal-flex pill tiles, DM Serif Display 20px apricot number, 8px uppercase hint label. Personality line below (DM Serif Display italic 11px, rotates).

Upgrade to Pro banner (Free only): surface-low bg, 1px plum-wash border, 32px icon container + copy + "See Pro" plum pill. Placed between stats and notifications. Removed entirely on Pro.

Notification preferences: surface-low card with 1px surface-high dividers between rows. Weekly digest row (label + day/time + pencil edit icon). Push reminders row (label + iOS toggle). Smart nudges row: Pro only, shown in Pro state.

Account section: Free: email + plan (Free) + "Upgrade to Pro" plum arrow row. Pro: email + plan (Pro, €4.99/mo) + "Manage subscription" + "Household sharing (1 of 3)".

Settings section: Dark mode toggle (off default) + Language + Region + Export my data (plum action). Same across Free/Pro.

Support section: Help, Privacy policy, Terms, Version (no chevron, no tap). Same across Free/Pro.

Sign out: Red centered text, no button shape. Confirmation in a bottom sheet (not shown in hi-fi).

No FAB on Profile tab. FAB conditionally hidden. Nav bar: Profile icon + label active in plum.
Flutter notes:
Shell: SafeArea + Column + Expanded(child: ListView) + fixed nav 84px. Avatar: CircleAvatar with GoogleFonts.notoSerif(fontStyle: FontStyle.italic) child. Plan state: UserPlan.isPro from RevenueCat CustomerInfo drives badge color, banner visibility, account row content, and smart nudges toggle. Toggles: Switch(activeColor: colorCta) or custom animated Container. Stats: aggregate query from Supabase. Version string: package_info_plus PackageInfo.fromPlatform(). Sign out: TextButton + showModalBottomSheet for confirmation. Export: Supabase function call + share sheet. Personality line: Supabase rotation pool, selected by userId.hashCode ^ weekNumber for weekly consistency. FAB: hidden via currentTab != TabIndex.profile in the shell scaffold.
