Shell + layout
Rule Shell pattern
Progress bar at 40% (S-03 = step 2 of 5). Full-width, edge-to-edge, outside SafeArea. Back button (ArrowLeft SVG, 44x44px tap target) returns to S-01. Scrollable content zone between back button and sticky CTA.
Rule Search bar
Magnifying glass icon + "Search items..." placeholder. White surface bg, 0.5px border, 10px radius. Live filter: filters visible items as user types. No separate search results screen. Search text clears when category filter changes.
Content
Rule Category filter strip
"All" pill always first, active by default. Category pills with colored dots. Single scrollable row (flex-wrap:nowrap; overflow-x:auto). Active pill: ink bg, cream text. Inactive pill: surface bg, 0.5px border, colorMidAccessible text. Each category pill has a 6px colored dot. Tapping a category filters the list below. Tapping "All" shows everything.
Decision Item selection
Items are tappable with check circles (20px diameter). Selected: colorCta fill + checkmark. Deselected: 1.5px border, transparent bg. CTA count updates on every toggle. Selected row: 1.5px CTA border + subtle CTA tint.
Decision Context carry-over from S-01
Items the user tapped on S-01 arrive pre-checked. In this mockup: Dishwasher and Car are pre-selected (2 items). Never ask users for information they already gave. If no items were selected on S-01, all items start unchecked.
Reminder count
Each item row shows a reminder count pill ("3 reminders", "2 reminders"). Shows the number of pre-loaded tasks nudgii has for that item. Does not show task names here: that detail lives on S-05 (review) and S-13 (task detail).
Section headers
Category section headers: 9px, 500 weight, 1.8px letter-spacing, uppercase, colorMidAccessible #6B6358. Items below each header are individual white cards with 6px gap. Cream between cards IS the visual separator.
Discovery + CTA
"Something else" row
Dashed border, plus icon. Discovery path for items not in the default browse list. Opens a deeper browse or search. Same pattern as S-01's "Something else" row. No check circle on this row.
Rule Sticky CTA
"Continue with X items" updates live. Count badge shows the total. Navigates to S-05 (review). Disabled state at 0 items: rgba(155,127,212,0.35) bg, rgba(250,248,255,0.5) text. Pressed: bg #7D62B3, scale 0.97, 100ms ease-out.
Changelog
v1.0 · 2026-03-24 · ADDITIVE
Initial hi-fi. Flat browsable item list with category filter strip, search bar, and pre-checked items from S-01. Replaces the earlier 2x2 category tile grid with a single scrollable list grouped by section headers.
Category filter strip. "All" pill added as first pill, active by default. Matches CLAUDE.md spec: S-03 and S-09 include "All" (S-01 does not). Tapping a category pill filters visible items. Tapping "All" shows everything.
Search bar. Live text filter above the category strip. Filters items by name as the user types. No separate search results screen.
Context carry-over. Items selected on S-01 (Dishwasher, Car) arrive pre-checked. CTA starts at "Continue with 2 items".
"Something else" row. Dashed border, plus icon, discovery path for items not in the default list.
Sticky CTA with live count. "Continue with X items" updates as user checks/unchecks. Disabled state at 0 selections.
Standard Flutter implementation block · S-03
Mockup base:    390px width (iPhone 15)
SafeArea:       top=false, progress bar outside SafeArea
Scroll:         SingleChildScrollView for content zone only
Bottom zone:    Sticky: fixed Column at bottom, not Spacer
Progress bar:   Full width, height 3px, outside SafeArea, 40% fill
Scaling:        All visual elements x (screenWidth/390).clamp(0.9, 1.15)
Special:        Category filter: SingleChildScrollView(horizontal) wrapping Row of FilterChip. Item rows: GestureDetector with AnimatedSwitcher on check circles (150ms). Search: TextField with onChanged callback filtering item list. Pre-checked items passed from S-01 via state. CTA disabled state via AnimatedContainer.
