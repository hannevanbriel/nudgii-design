Design rationale
Status grouping Things 3
"Could use you" floats to top. Empty section = instant calm. Power users (Arne) scan one section and are done. Status is the organizing principle, not category.
"Could use you", not "overdue" Voice
Tasks have time windows, not deadlines. "Refill dishwasher salt" is due every 6-8 weeks, not on a specific day. The word "overdue" creates guilt. nudgii reframes: the item needs you, you haven't failed it.

Section header: "Could use you" (item is the subject)
Task status: "1 task waiting" (factual, no blame)
Personality: "Salt was last topped up 10 weeks ago." (time since, not time past due)

The apricot color on the ring and task count is the only urgency signal. No "overdue" badge, no accent line, no red.
Category wash on cards Apple Home
Items sit on their category wash color. Tasks on Today sit on neutral surface-low. This is the primary visual cue that these are objects, not actions. "Could use you" items get stronger wash (Apple Home: device tiles shift color with state). No left-border accent: the wash IS the signal.
Progress ring Things 3
32px ring shows tasks on track / total tasks as a ratio (e.g. "2/3", "4/5"). Not a percentage: a ratio is instantly clear without needing to know the timeframe.

Ring color = category color (home=dusty rose, vehicle=sage-slate, garden=sage, subs=lavender). The only exception: items in "Could use you" use apricot to signal attention.

What counts as a task: Active tasks in the current maintenance cycle. Recurring tasks (e.g. monthly) count as 1 task, not 12. A task is "on track" when it's within its time window.
Nicknames Opt-in
Default name = generic item type ("Dishwasher", "Boiler", "Car"). This is what the screen shows.

Nicknames are opt-in. After adding an item, nudgii offers a short nickname as a suggestion chip: "Added. Want to give it a nickname?" Most users skip. Those who tap it get a fun extra.

Examples: "Dishy" (dishwasher), "Bubs" (boiler), "Wheels" (car), "Ollie" (olive tree), "Flix" (Netflix)

Rules: one word preferred, two max. Obvious, not clever. Never the default. Offered once, not pushed. After AI scan, brand-based: "The Bosch."

Display: item name in Noto Serif 15px. Different typeface from task names on Today (Manrope 14px), reinforcing that these are objects, not actions.
Personality on attention + seasonal only Greg
On-track items stay clean (name + meta + ring). Only items that need conversation show the italic personality line. This creates information density where it matters and calm where it doesn't.
Status group logic
Priority order Locked
Items appear in exactly one group. Priority determines which:

1. Could use you (apricot dot): any task past its recommended window. Always wins.
2. Seasonal (category dot): a calendar-based window is currently open, task not yet done.
3. On track (sage dot): all tasks within their recommended window.

Groups are live queries, not permanent labels. Items move between groups as tasks change state.
Seasonal behavior Locked
Seasonal = a calendar-based time window is currently open. Not a permanent label.

Example: olive tree
Jan: On track (no window open)
Mar: Seasonal (spring pruning window opens)
Apr: user prunes, marks done → On track
Nov: Seasonal again (winter protection window)

The Seasonal section can be empty for months and have 4 items in October. If a seasonal window closes without completion, the item moves to "Could use you."
Task types Dev spec
Three types, each counts as 1 task in the progress ring:

Interval-based: "every 6-8 weeks." On track = within window. Waiting = past window. Ring shows tasks on track / total.
Calendar-based (seasonal): "March to May." Triggers Seasonal group when window opens. Moves to "could use you" if window closes undone.
One-time: "register warranty." Done = gone from ring count.

Recurring tasks count as 1 task, not N occurrences. A monthly task is 1 task that's either on track or waiting.
Interaction spec
Category filter Locked
Active states (locked 2026-04-03):
"All" = plum fill + cream text
Home = dusty rose #C09088 + cream
Vehicle = sage-slate #8E9A94 + cream
Garden = sage #8C9B80 + cream
Subs = lavender #A69BBF + cream

Behavior: Single select. Tapping a category shows only items in that category. Empty status groups hide entirely. Tapping "All" resets. Single scrollable row, no wrap.
Go Pro meter Free tier
Fixed above nav bar, free users only. Shows "X of 10 free" + dot meter (plum filled, surface-high empty) + "Go Pro" link. Hidden for Pro users. Never blocks, never nags. Items 11+ are paused (no reminders) but visible with Pro badge.
Item tap Navigation
Tapping an entity card navigates to S-17 item detail. Card shows :active state (scale 0.98) on press. No swipe gestures on My Stuff (swipe-to-complete is Today tab only).
Dark mode Complete
Full dark mode via [data-theme="dark"]. Warm dark surfaces (#1C1B18). Category wash at 10-18% opacity (stronger for "could use you"). Plum lightened to #B8A0E8. Ring tracks use rgba(255,255,255,0.08). No cold/blue tints.
Wash intensity Dev spec
Category wash on cards:
Normal: 14% opacity (e.g. rgba(192,144,136,0.14))
"Could use you": 22% opacity (rgba(192,144,136,0.22))
The stronger wash is the only visual urgency signal. No left-border accent, no badge, no red.
Entity vs. task: 7 visual differences
How users know they're looking at items, not tasks
1. Category wash background (tasks: neutral)
2. Noto Serif name (tasks: Manrope)
3. Progress ring in category color (tasks: no ring)
4. 44px icon circle (tasks: 40px, tighter)
5. 14px rounded corners (tasks: 12px)
6. 16px padding (tasks: 10-12px)
7. Italic personality line (tasks: never)

None of these is dramatic alone. Together they create a distinct visual rhythm: items feel spacious and owned, tasks feel compact and actionable.
Persona fit
Arne Power user
Scans "Could use you" header, sees 1 item, taps it or moves on. Done in 3 seconds. Status grouping is built for him.
Martijn Primary
Opens My Stuff weekly. Progress rings give instant health check. Empty attention section = "everything's fine" dopamine hit. Category wash colors make the screen feel personal.
Lena Learning
Personality lines teach her about her things ("Pruning window is open. She's ready."). Seasonal section introduces concepts she doesn't know yet. The screen is informative without being overwhelming.
Sara Organizer
Progress rings satisfy her completionist instinct. She wants to see 100% on everything. Category filter lets her browse by domain. The system mirrors how she already thinks about her possessions.
Open questions
Scale at 15+ items
With 15 items, the on-track section gets long. Options: collapse sections (Things 3 pattern), show count-only in collapsed mode, or add a "show all" button after 5 items. Test with Arne persona data.
Photo support (v2)
Plant apps lead with user photos. nudgii could show an AI-scan photo as the icon background (from camera input). v1 ships with category icons. v2 adds optional photo from scan, displayed inside the 44px circle.

Changelog
v1.0 · 2026-04-03 · BREAKING
New screen. Replaces flat list layout from s09-mystuff-hifi.html with status-grouped entity cards.

Visual changes: Status grouping (Could use you > Seasonal > On track). Entity cards with category wash bg, 44px icons, Noto Serif names, progress rings in category colors, ratio labels. No separator lines between groups. Category-colored filter pills.

Language: "Could use you" replaces "Needs attention." "1 task waiting" replaces "1 overdue." Urgency language ladder locked. Personality names as default item names ("The workhorse", "Old reliable").

Interaction: Filter is functional (single select, groups hide when empty). Item cards tap through to S-17 detail pages.
FLUTTER NOTES
Mockup base: 390x844 iPhone 15. SafeArea + Column + Expanded shell. Status groups are filtered ListViews, not static sections. Progress ring: CustomPainter with category color stroke. Filter: single-select chip row, updates list query. Go Pro meter: positioned above nav, free tier only. Entity cards: GestureDetector → Navigator.push to S-17. Dark mode: full token swap via ThemeData.
