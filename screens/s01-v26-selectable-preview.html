Selectable item rows new
Tap any row to toggle selected. Multiple allowed. Nothing required — buttons work either way.

Selected carries context:
· Path A (voice): nudgii pre-loads "I see you have a dishwasher and winter tyres — what else?"
· Path B (browse): those categories open first

border 0.5px rgba(26,22,18,0.08) · white bg · check circle: empty outline
Selected
border 1px rgba(155,127,212,0.35) · bg rgba(155,127,212,0.04) · check: colorCta fill + white tick
Animation
150ms ease-out on border + background
Flutter
AnimatedContainer 150ms + GestureDetector onTap → setState(selected toggle)

"Something else" row new
Always 4th. Dashed border signals "different action." Tapping it routes directly to Path B (browse categories) — no voice, no search.

Solves the edge case: someone with no car, no garden, renting — none of the three examples fit. They always have a way in.

Button zone — unchanged
CTAs are always active — selection is optional context, not a required gate. Buttons never change label or state based on selection.

Nothing selected = CTAs work exactly as before. Something selected = same CTAs, with pre-loaded context passed behind the scenes.

Are the 3 example items fixed (always Dishwasher · Winter tyres · Olive tree) or seasonal/locale-aware (pulled from a small curated set based on month + region)?
