Shell + layout
Rule Progress bar
Full-width, edge-to-edge, outside SafeArea. 3px height, colorCta #9B7FD4 fill at 20% (S-01 = step 1 of 5). Track: rgba(26,22,18,0.08). Never segmented, never dots. Flutter: LinearProgressIndicator inside ClipRRect(borderRadius:100px).
Rule Scroll + sticky zones
Content zone scrolls (ii mark through task rows). Button zone is fixed at bottom, outside scroll area. Flutter: Column with Expanded(child: SingleChildScrollView) + fixed bottom Column. Never a Spacer to push buttons down.
Rule ii mark + free badge
ii mark: inline SVG, 18x20px. Dark-on-light variant (CTA purple left, plum right, apricot dot). Free badge: sage-lt #EAF3DE bg, sage #3B6D11 text, pill shape with 5px sage dot. Copy: "Free · no credit card".
Content
Headline + subtitle
DM Serif Display 33px, "maintain itself." in colorApricotDk #C8784A italic. Subtitle: "Add what you own. We'll remind you when it matters." in 13px italic DM Sans, colorMid. "Ready in 2 minutes" badge: clock icon + text in subtle pill.
Category strip
4 pills: Home, Vehicle, Garden, Subscriptions. No "All" pill on S-01 (decorative, takes focus). "All" kept on S-03 and S-09 where it filters. Single scrollable row, flex-wrap:nowrap. Each pill has a colored 7px dot.
Decision Interactive item rows
Items are tappable with check circles. All rows start unselected. "For example, tap what applies to you:" microcopy provides affordance. Pre-selected items carry into S-02 as confirmed chips. Real app: subtle pulse animation on check circles at first load.
Decision Task names, not item names
Rows show task descriptions ("Clean dishwasher filter") not item names ("Dishwasher"). This previews what nudgii actually does: remind you of specific maintenance tasks. Category context ("Home") + cadence pill ("every 6-8 wks") complete the picture.
Decision CV ketel over hedge trim
95%+ of BE/NL homes have central heating. Annual boiler service is safety-critical and universally forgotten. Exactly the nudgii value prop in one row.
Button zone
Decision Two paths, no mic icon
Path A "Tell me what you have" leads to S-02 (voice/type/scan). No mic icon in the button: Path A covers voice, text, and scan input. A mic icon was misleading (implied voice-only). Path B "Show me what I can track" leads to S-03 (browse categories). Path C ghost button removed: two clear paths is better than three + a footnote. "Something else" row already covers discovery.
Rule Button hierarchy
Path A: colorCta fill, 50px height, pill, shadow. Path B: transparent, 1.5px border, 46px height. Always vertically stacked, never side-by-side. Sentence case. Divider: "How do you want to start?" with lines.

Standard Flutter implementation block · S-01
Mockup base:    390px width (iPhone 15)
SafeArea:       top=false, progress bar outside SafeArea
Scroll:         SingleChildScrollView for content zone only
Bottom zone:    Sticky: fixed Column at bottom, not Spacer
Progress bar:   Full width, height 3px, outside SafeArea, 20% fill
Scaling:        All visual elements x (screenWidth/390).clamp(0.9, 1.15)
Special:        Task rows are tappable (GestureDetector). Pre-selected items pass to S-02 via state. Check circles animate with 150ms spring (AnimatedSwitcher). "Something else" row navigates to S-03.
