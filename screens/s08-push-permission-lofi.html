Annotations — S-08
1
Illustration / icon One screen
A single visual that communicates "nudges" — a calendar icon, a bell, or something nudgii-specific. Not a stock illustration. Decide in hi-fi. Keeps the screen short and focused.
2
Headline + body
Headline (DM Serif Display): "Mis niks." or "Eén seintje per taak." Body (DM Sans): one sentence explaining the weekly digest and day-of nudge. Copy must feel earned — user just set up their whole home, they trust nudgii. Don't be defensive about asking.
3
3 benefit rows
Icon + title + one-line description. Locked benefits:

· Weekoverzicht elke maandag
· Eén push op de dag zelf
· Stil op andere dagen

"Stil op andere dagen" is the most important line — it reduces permission anxiety.
4
nudgii CTAs Fires iOS dialog
Primary: "Ja, stuur me nudges" — tapping fires the iOS native permission dialog immediately.

Secondary: "Misschien later" — skips to dashboard without requesting permission. User can enable later from settings.
5
iOS native dialog System UI
Appears over the nudgii screen after primary tap. Text: "nudgii wil je berichten sturen" — set in app's Info.plist NSUserNotificationUsageDescription. Cannot be customised beyond the usage string. Allow / Niet toestaan.
Why this screen exists

iOS only shows the native permission dialog once. If the user taps "Niet toestaan", push is disabled permanently until they go to Settings. nudgii's pre-prompt gives the user the context they need to make an informed choice — and dramatically increases opt-in rate.

~40%

Layout decision locked

No back button — user has just completed onboarding and this is the final step. Skipping via "Misschien later" is the only exit.

Screen fires after the onboarding completion screen (S-07), before the user sees the dashboard for the first time.

Copy direction: "Stil op andere dagen" is the most persuasive line. Lead with it in the benefit list.
