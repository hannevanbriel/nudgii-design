1. Screen overview

What this screen does

S-03 is the Path B onboarding screen. The user browses items by category using a tile grid and taps to select what they own. In overview mode ("All" filter active), each category shows a horizontally scrollable row of tiles. When the user taps a category chip, the view collapses into a full 3-column grid showing only that category. A selected pill tracks the count. The CTA copy changes as selections accumulate. The progress bar grows with each selection.

  • Source hi-fi: s03-browse-categories-hifi.html
  • Source prototype: s03-browse-categories-prototype.html
  • Progress bar fill: 40% base + 8% per selected item, clamped at 80%
  • No tab bar. This is an onboarding screen.
  • Back button in the top-left of the content zone (not a native AppBar).
  • Navigation: Back goes to S-02. CTA taps go to S-04 (Review screen).

2. State model

2.1 Riverpod state class

// Use Riverpod with a StateNotifier for the screen state.

enum CatFilter { all, home, vehicle, garden, subscriptions }

@immutable
class S03State {
  final CatFilter activeFilter;
  final String searchQuery;
  final Set<String> selectedIds;

  const S03State({
    this.activeFilter = CatFilter.all,
    this.searchQuery = '',
    this.selectedIds = const {},
  });

  S03State copyWith({
    CatFilter? activeFilter,
    String? searchQuery,
    Set<String>? selectedIds,
  }) => S03State(
    activeFilter: activeFilter ?? this.activeFilter,
    searchQuery: searchQuery ?? this.searchQuery,
    selectedIds: selectedIds ?? this.selectedIds,
  );

  int get selectionCount => selectedIds.length;
  bool isSelected(String id) => selectedIds.contains(id);
}

class S03Notifier extends StateNotifier<S03State> {
  S03Notifier() : super(const S03State());

  void setFilter(CatFilter filter) => state = state.copyWith(activeFilter: filter);

  void toggleItem(String id) {
    final updated = Set<String>.from(state.selectedIds);
    if (updated.contains(id)) {
      updated.remove(id);
    } else {
      updated.add(id);
    }
    state = state.copyWith(selectedIds: updated);
  }

  void setSearch(String q) => state = state.copyWith(searchQuery: q);
}

final s03Provider = StateNotifierProvider<S03Notifier, S03State>(
  (_) => S03Notifier(),
);

3. Screen shell

class BrowseCategoriesScreen extends ConsumerStatefulWidget {
  const BrowseCategoriesScreen({super.key});

  @override
  ConsumerState<BrowseCategoriesScreen> createState() =>
      _BrowseCategoriesScreenState();
}

class _BrowseCategoriesScreenState
    extends ConsumerState<BrowseCategoriesScreen> {

  bool _hintDone = false; // swipe hint runs once per session

  @override
  void initState() {
    super.initState();
    // Trigger swipe hint after first frame
    WidgetsBinding.instance.addPostFrameCallback((_) {
      if (!_hintDone) _runSwipeHint();
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: colorCream,
      body: SafeArea(
        top: false, // progress bar bleeds under status bar
        child: Column(
          children: [
            _buildProgressBar(),       // outside SafeArea, full-width
            Expanded(
              child: Column(
                children: [
                  _buildScrollableContent(), // scrolls
                ],
              ),
            ),
            _buildBottomZone(),        // fixed, does NOT scroll
          ],
        ),
      ),
    );
  }
}
  • Progress bar is the very first widget, rendered before SafeArea kicks in, so it bleeds edge-to-edge under the status bar.
  • The content zone above the CTA scrolls. The CTA zone is always visible.
  • No SingleChildScrollView wrapping the entire screen. Only the content zone scrolls.

4. Progress bar

// Base fill: 40%. Each selection adds 8%, clamped at 80%.
// Animates on every selection/deselection.

Widget _buildProgressBar() {
  final selCount = ref.watch(s03Provider).selectionCount;
  final fillFraction = (0.40 + selCount * 0.08).clamp(0.0, 0.80);

  return SizedBox(
    height: 3,
    child: LayoutBuilder(
      builder: (context, constraints) {
        return Stack(
          children: [
            // Track
            Container(
              width: constraints.maxWidth,
              height: 3,
              color: colorCream, // same as bg — track is invisible
            ),
            // Fill
            AnimatedContainer(
              duration: const Duration(milliseconds: 500),
              curve: Curves.easeInOut,
              width: constraints.maxWidth * fillFraction,
              height: 3,
              decoration: BoxDecoration(
                color: colorCta,
                borderRadius: BorderRadius.only(
                  topRight: Radius.circular(2),
                  bottomRight: Radius.circular(2),
                ),
              ),
            ),
          ],
        );
      },
    ),
  );
}

5.2 Individual chip widget

Widget _buildChip(CatFilter filter, String label, CatFilter active) {
  final isActive = filter == active;

  // Colors per filter
  final Map<CatFilter, Color> chipBgColor = {
    CatFilter.all:           const Color(0x1F9B7FD4), // rgba(155,127,212,0.12)
    CatFilter.home:          const Color(0x21C87850), // rgba(200,120,80,0.13)
    CatFilter.vehicle:       const Color(0x214A82A0), // rgba(74,130,160,0.13)
    CatFilter.garden:        const Color(0x215A9060), // rgba(90,144,96,0.13)
    CatFilter.subscriptions: const Color(0x218060A8), // rgba(128,96,168,0.13)
  };

  final Map<CatFilter, Color> chipTextColor = {
    CatFilter.all:           colorCta,                // #9B7FD4
    CatFilter.home:          catHome,                 // #C87850
    CatFilter.vehicle:       catVehicle,              // #4A82A0
    CatFilter.garden:        catGarden,               // #5A9060
    CatFilter.subscriptions: catSub,                  // #8060A8
  };

  final Map<CatFilter, Color> chipBorderColor = {
    CatFilter.all:           colorCta,
    CatFilter.home:          catHome,
    CatFilter.vehicle:       catVehicle,
    CatFilter.garden:        catGarden,
    CatFilter.subscriptions: catSub,
  };

  return GestureDetector(
    onTap: () => ref.read(s03Provider.notifier).setFilter(filter),
    child: AnimatedContainer(
      duration: const Duration(milliseconds: 150),
      curve: Curves.easeOut,
      padding: const EdgeInsets.symmetric(horizontal: 12, vertical: 5),
      decoration: BoxDecoration(
        color: chipBgColor[filter],
        borderRadius: BorderRadius.circular(100),
        border: isActive
          ? Border.all(color: chipBorderColor[filter]!, width: 1)
          : Border.all(color: Colors.transparent, width: 1),
      ),
      child: Text(
        label,
        style: GoogleFonts.dmSans(
          fontSize: 11,
          fontWeight: FontWeight.w500,
          color: chipTextColor[filter],
        ),
      ),
    ),
  );
}

// IMPORTANT: The border must always be present (transparent when inactive)
// to prevent layout shift when the border appears on selection.
// Use Border.all(transparent, width:1) for inactive state.

6. Tile widget

// Each tile is 88px wide (in grid: adapts to 1/3 column width).
// 14px border-radius. Category color at 12% opacity background.
// Selected: category-color border 1.5px (NOT purple).
// Press: scale(0.93). Hover/focus: scale(1.03).

Widget _buildTile(CategoryItem item, bool isSelected) {
  final catColor = _categoryColor(item.category); // e.g. catHome

  return GestureDetector(
    onTap: () => ref.read(s03Provider.notifier).toggleItem(item.id),
    child: AnimatedContainer(
      duration: const Duration(milliseconds: 150),
      curve: Curves.easeOut,
      width: 88, // in grid mode: set to null and use Column parent
      padding: const EdgeInsets.fromLTRB(6, 13, 6, 9),
      decoration: BoxDecoration(
        color: catColor.withOpacity(0.12),
        borderRadius: BorderRadius.circular(14),
        border: Border.all(
          color: isSelected ? catColor : Colors.transparent,
          width: 1.5,
        ),
      ),
      child: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          Text(item.emoji, style: const TextStyle(fontSize: 22)),
          const SizedBox(height: 6),
          Text(
            item.label,
            textAlign: TextAlign.center,
            style: GoogleFonts.dmSans(
              fontSize: 9.5,
              fontWeight: FontWeight.w500,
              color: colorInk,
              height: 1.2,
            ),
          ),
        ],
      ),
    ),
  );
}

Color _categoryColor(CatFilter category) {
  switch (category) {
    case CatFilter.home:          return catHome;
    case CatFilter.vehicle:       return catVehicle;
    case CatFilter.garden:        return catGarden;
    case CatFilter.subscriptions: return catSub;
    case CatFilter.all:           return colorCta;
  }
}
  • Selected border is always the category color (never purple / colorCta). This clearly communicates which category the item belongs to.
  • In 3-col grid mode, set width: null — the GridView handles sizing. Keep all other styles identical.
  • Minimum tap target: 44x44px. The tile padding should ensure the combined height is at least 44px.

7.1 Category section

// In "All" mode, each category renders as:
// 1. Section header (section label + colored dot)
// 2. Horizontal ScrollView of tiles
// 3. A peek arrow indicating more content to the right

Widget _buildCategorySection(CatFilter cat, List<CategoryItem> items) {
  final catColor = _categoryColor(cat);
  final catLabel = _categoryLabel(cat); // 'HOME', 'VEHICLE', etc.
  final state = ref.watch(s03Provider);

  // Filter by search query
  final visible = items.where((i) =>
    i.label.toLowerCase().contains(state.searchQuery.toLowerCase())
  ).toList();

  if (visible.isEmpty) return const SizedBox.shrink();

  return Column(
    crossAxisAlignment: CrossAxisAlignment.start,
    children: [
      // Section header
      Padding(
        padding: const EdgeInsets.only(top: 8, bottom: 5),
        child: Row(
          children: [
            Container(width: 6, height: 6,
              decoration: BoxDecoration(
                color: catColor,
                shape: BoxShape.circle,
              ),
            ),
            const SizedBox(width: 5),
            Text(
              catLabel,
              style: GoogleFonts.dmSans(
                fontSize: 9,
                fontWeight: FontWeight.w600,
                letterSpacing: 1.3,
                color: colorMidAccessible,
              ),
            ),
          ],
        ),
      ),
      // Horizontal tile row
      SingleChildScrollView(
        scrollDirection: Axis.horizontal,
        padding: const EdgeInsets.only(bottom: 5, top: 3),
        child: Row(
          children: [
            ...visible.map((item) =>
              Padding(
                padding: const EdgeInsets.only(right: 7),
                child: _buildTile(item, state.isSelected(item.id)),
              )
            ),
            // Peek arrow (always last, indicates more content)
            if (visible.length > 3)
              Padding(
                padding: const EdgeInsets.only(left: 2),
                child: Icon(PhosphorIcons.caretRight,
                  size: 13,
                  color: colorInk.withOpacity(0.20),
                ),
              ),
          ],
        ),
      ),
    ],
  );
}

7.2 Swipe hint animation

// On first load (before user has scrolled), animate the Home row
// scrolling 52px to the right, pause 520ms, then snap back.
// This hints that rows are horizontally scrollable.
// Uses _hintDone flag: runs once per app session.

// Each category section needs its own ScrollController.
final ScrollController _homeRowController = ScrollController();

Future<void> _runSwipeHint() async {
  if (_hintDone) return;
  _hintDone = true;

  // Small delay so the row is rendered before animating
  await Future.delayed(const Duration(milliseconds: 600));
  if (!mounted) return;

  // Scroll right 52px
  await _homeRowController.animateTo(
    52,
    duration: const Duration(milliseconds: 400),
    curve: Curves.easeInOut,
  );

  // Pause
  await Future.delayed(const Duration(milliseconds: 520));
  if (!mounted) return;

  // Snap back
  await _homeRowController.animateTo(
    0,
    duration: const Duration(milliseconds: 300),
    curve: Curves.easeOut,
  );
}

// Attach _homeRowController to the Home category's SingleChildScrollView.
// Do NOT attach the same controller to other rows.

// IMPORTANT: Destroy the swipe hint if the user manually scrolls first.
// Add a listener to _homeRowController that sets _hintDone = true
// on user-initiated scroll, and cancel the Future if still pending.

8. Category mode: 3-column grid

// When any category chip other than "All" is tapped,
// switch to a full 3-column grid showing only that category's items.

Widget _buildCategoryGrid(CatFilter cat, List<CategoryItem> items) {
  final state = ref.watch(s03Provider);
  final visible = items.where((i) =>
    i.label.toLowerCase().contains(state.searchQuery.toLowerCase())
  ).toList();

  return GridView.builder(
    shrinkWrap: true,
    physics: const NeverScrollableScrollPhysics(), // parent handles scroll
    gridDelegate: const SliverGridDelegateWithFixedCrossAxisCount(
      crossAxisCount: 3,
      crossAxisSpacing: 7,
      mainAxisSpacing: 7,
      childAspectRatio: 0.85, // approx 88px wide tile, ~104px tall
    ),
    itemCount: visible.length,
    itemBuilder: (_, i) => _buildTile(visible[i], state.isSelected(visible[i].id)),
  );
}

// Switch between overview and grid mode based on activeFilter:
Widget _buildItemContent(List<CategoryItem> allItems) {
  final filter = ref.watch(s03Provider).activeFilter;

  if (filter == CatFilter.all) {
    // Overview: one horizontal row per category
    return Column(
      crossAxisAlignment: CrossAxisAlignment.start,
      children: CatFilter.values
        .where((c) => c != CatFilter.all)
        .map((c) => _buildCategorySection(
              c,
              allItems.where((i) => i.category == c).toList(),
            ))
        .toList(),
    );
  } else {
    // Single category: 3-col grid
    return _buildCategoryGrid(
      filter,
      allItems.where((i) => i.category == filter).toList(),
    );
  }
}

11.2 Flutter implementation

String _ctaCopy(int count) {
  if (count == 0) return 'Tick something first';
  if (count == 1) return 'Good start, continue';
  if (count == 2) return 'Nice, keep going or continue';
  return 'Looking good, continue';
}

Widget _buildBottomZone() {
  final count = ref.watch(s03Provider).selectionCount;
  final isEnabled = count > 0;

  return Padding(
    padding: const EdgeInsets.fromLTRB(20, 8, 20, 20),
    child: AnimatedSwitcher(
      duration: const Duration(milliseconds: 180),
      transitionBuilder: (child, animation) =>
        FadeTransition(opacity: animation, child: child),
      child: GestureDetector(
        key: ValueKey(count > 3 ? '3+' : count), // trigger rebuild per state
        onTap: isEnabled ? _onContinue : null,
        child: AnimatedContainer(
          duration: const Duration(milliseconds: 150),
          curve: Curves.easeOut,
          width: double.infinity,
          padding: const EdgeInsets.symmetric(vertical: 13),
          decoration: BoxDecoration(
            color: isEnabled ? colorCta : colorInk.withOpacity(0.08),
            borderRadius: BorderRadius.circular(100),
            boxShadow: isEnabled
              ? [BoxShadow(
                  color: colorCta.withOpacity(0.35),
                  blurRadius: 18,
                  offset: const Offset(0, 4),
                )]
              : [],
          ),
          alignment: Alignment.center,
          child: Text(
            _ctaCopy(count),
            style: GoogleFonts.dmSans(
              fontSize: 13,
              fontWeight: FontWeight.w500,
              color: isEnabled ? colorCtaFg : colorMid,
            ),
          ),
        ),
      ),
    ),
  );
}

void _onContinue() {
  final selected = ref.read(s03Provider).selectedIds;
  context.push('/onboarding/review', extra: {'selectedIds': selected});
}
  • AnimatedSwitcher with FadeTransition handles the 180ms label crossfade when the copy changes.
  • The key on the inner widget must change per copy state, not per count — otherwise the fade fires too often (at count 4, 5, 6 etc the copy stays "Looking good, continue").
  • Bottom zone is outside Expanded. It does NOT scroll. It stays pinned to the bottom at all times.
  • Bottom padding: 20px minimum. Add MediaQuery.of(context).padding.bottom on devices with home indicator.

14. Edge cases

Empty search results
If the search query returns no items in any category (overview mode) or no items in the selected category (grid mode), show a centered empty state: "Nothing matched. Try a different word." in 12px DM Sans italic, colorMid. Do not show any category headers when all sections are empty.
Item data loading
While category_items loads from Supabase, show a skeleton state: grey placeholder tiles in the same tile shape, using Shimmer effect. Do not block the UI. If loading fails, show the error card component (full-border + red-lt bg + icon-chip) above the chip bar.
Swipe hint race condition
If the user taps a tile or scrolls before the swipe hint fires (600ms delay), cancel the hint. Attach a scroll listener to _homeRowController that sets _hintDone = true on any user gesture, before the Future.delayed fires.
Large selection counts
The selected pill count badge is 16x16px. For counts above 99, show "99+" in a slightly smaller font (7px). The pill width will expand naturally as the count label grows.
Free tier limit — no blocking during onboarding
Never block or warn the user during S-03 selection, even if they select more than 10 items. The 10-item free tier limit is enforced on the dashboard only. Items 11+ get a "Pro" pill on the dashboard and are paused. Let users select freely here.
Filter switch with active search
When the user switches from a category filter back to "All", preserve the search query. The search filters across all categories in overview mode. Only clear the search field if the user explicitly taps the clear (X) icon.
Back navigation with selections
If the user presses back (to S-02) with items already selected, preserve the selection in S03State. If they return to S-03, their previously selected tiles should still be highlighted. Do not clear state on back navigation.
