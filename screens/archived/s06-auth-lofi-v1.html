Screen layout
AStatus bar
Standard iOS status bar. 20px height. Transparent over cream.
BProgress bar — 80% (step 4 of 5)
Continuous bar, 3px, full-width edge-to-edge. Fill: colorCta #9B7FD4. Track: rgba(26,22,18,0.08).
S-06 = 80%. Zeigarnik effect — user can see they're almost done. Progress is visibly earned after completing S-07a (AHA schedule reveal).
CContent area — vertically centered
Shell pattern: flex:1 — fills all space between progress bar and sticky bottom.
ii-mark centered (left=ink, right=cta 45% opacity, dot=apricot-dk).
"Almost done." — DM Serif Display 22px. Deferred auth = user has invested 60s and seen their tasks. Not "Create an account" — that sounds like work.
Subtitle: "Save your schedule with a free account." — DM Sans 10px, colorMid, centered.
D · ESticky bottom — SSO buttons
Apple always first — iOS HIG requirement. Do not deviate.
Both buttons: 44px height · white bg · border rgba(26,22,18,0.18) · border-radius 100px · 6px gap between them.
Below Google: text link "or use email instead" — colorCta, no button style, no divider line.
Tapping reveals email input + CTA inline (Zone G+H expands above the buttons). SSO buttons slide up. No new screen, no navigation.
G · HEmail expand state
Revealed inline when user taps "use email instead". Pushes up from above the SSO buttons.
Zone G: email input, 44px, border-radius 10px, placeholder "email address".
Zone H: CTA "Send magic link" — 46px, colorCta bg, pill-radius, colorCtaFg text.
On send: inline confirmation replaces CTA — "Check your inbox." — no new screen.
Magic link = no password to create or remember. Lowest friction for the email path.
ERRAuth failure state
Inline error card below the CTA — no new screen.
Background: red-lt #FCEBEB · full border rgba(163,45,45,0.18) — no border-left accent.
Copy: "Sign in failed. Try again — your data is saved."
Onboarding state is stored locally — items are never lost on auth failure.
NOTEReassurance + privacy
"Free account · no credit card required" — 8px DM Sans, colorMid, centered. Reduces hesitation at account creation.
"Terms of service" text link below — 8px, underline.

iOS HIG requirement. Apple login always above Google on iOS. Do not deviate.
Magic link = no password
Lowest friction for the email path. No password to create, no password to remember.
State never lost

Reassurance line
"Free account · no credit card required" shown below magic link CTA. Reduces hesitation at account creation moment.

Changelog
v1.1 · BREAKING
Progress indicator replaced. Old 3-dot system removed. New: continuous bar, 3px height, full-width edge-to-edge, colorCta #9B7FD4 fill at 80% (S-06 = step 4 of 5).
Flutter: LinearProgressIndicator inside ClipRRect(borderRadius:100px), value:0.8. Remove old Row of dot Containers entirely.
v1.1 · BREAKING
Button layout restructured to sticky bottom zone. Previously: all zones stacked inline from top, large empty bottom half. Now: content area (ii mark + headline) uses flex:1 and is vertically centred. SSO buttons + email toggle are in a fixed bottom Column anchored to screen bottom.
Flutter: Column with Expanded(child: Center(child: headline content)) + fixed bottom Column (Apple btn, Google btn, text toggle). Not Spacer. Not SingleChildScrollView for the whole screen — only the content zone if content ever overflows.
v1.1 · BREAKING
Email path changed to inline toggle. Previously: "or" divider + always-visible email input + magic link CTA. Now: "or use email instead" text link (colorCta) below Google button. Tapping reveals email input + CTA inline above SSO buttons — no navigation, no new screen.
Flutter: AnimatedContainer or Visibility widget wrapping email input + CTA. Expand from height:0 to full height on tap, 200ms ease-in-out. SSO buttons stay visible below.
v1.0 · ADDITIVE
Auth failure state. Inline error card below magic link CTA on failed auth. Red-lt background, full border (no border-left accent), icon-chip 28×28px. Copy: "Sign in failed. Try again — your data is saved." Onboarding state is preserved locally — items never lost.
Flutter: Visibility(visible: authFailed) wrapping error card widget. Never navigate away. Keep onboarding state in local storage or provider.
