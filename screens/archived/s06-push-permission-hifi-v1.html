Push permission

Final onboarding step before the dashboard. Shows a concrete upcoming reminder date to make the value of notifications tangible. iOS only shows the native dialog once, so this screen dramatically increases opt-in rate. Two states: default (pre-prompt) and iOS dialog overlay.

Shell + layout
Rule Centered content pattern
Vertically centered content zone + sticky bottom CTAs. No scroll needed, content is short. Progress bar at 100% (final onboarding step). No back button (forward-only flow, iOS swipe-back gesture handles navigation).
Decision Pre-prompt before iOS dialog
iOS only shows the native permission dialog once. If the user taps "Don't Allow", push is permanently disabled until they manually go to Settings. The pre-prompt gives context and dramatically increases opt-in: ~40% without vs ~70% with a good pre-prompt.
Content
Bell icon
Bare Phosphor bell outline, 48px, stroke colorCta (#9B7FD4) at 60% opacity, stroke-width 1.6. No container background, no border. Just the icon floating. Lighter than the old boxed version, less "permission screen" and more "helpful nudge."
Decision Dynamic date in headline
DM Serif Display 28px: "We'll remind you on Oct 28." The date is dynamic: Flutter pulls the earliest upcoming reminder date from the user's newly created schedule. "Oct 28" is a placeholder in the mockup. The italic date in colorApricotDk (#C8784A) makes the value of notifications concrete and personal, not abstract.
Subline
13px italic, colorMid, centered, max-width 260px. "We only reach out when something actually needs your attention." Answers the user's unspoken fear: "Will this spam me?" Replaced the 3-benefit-row pattern with a single honest line.
CTAs + navigation
Primary CTA
"Turn on notifications" (colorCta bg, pill shape, 50px height). Tapping triggers the iOS native permission dialog immediately. Pressed state: bg #7D62B3, scale 0.97, 100ms ease-out.
Ghost CTA
"Maybe later" (transparent bg, colorMid text, no border, no background, 40px height). Skips to dashboard without requesting permission. Never blocks onboarding. User can enable later from profile settings.
Settings note
"You can always turn this on later in settings." (11px, colorMid, centered). Reduces pressure on the decision. Shown below the ghost CTA.
Decision Both paths lead to dashboard
If "Turn on notifications": iOS native dialog fires. Allow or deny, both proceed to the dashboard.
If "Maybe later": skips iOS dialog entirely, proceeds to dashboard.

"Maybe later" never blocks. No guilt, no retry, no dark pattern. The user made their choice.
Decision iOS native dialog
After "Turn on notifications" tap, iOS shows its native dialog: "nudgii Would Like to Send You Notifications". This text is set in the app's Info.plist (NSUserNotificationUsageDescription). Cannot be customized beyond the app name. Two system buttons: "Don't Allow" / "Allow".

If allowed: brief success (checkmark animation on bell icon, 400ms), then auto-advance to dashboard.
If denied: screen dismisses, proceeds to dashboard. No error, no retry, no guilt.
