Onboarding review

The user reviews items collected during onboarding (from any path). Each item card expands to show tasks, a scan button for brand/model detection, and a "last done" picker that adjusts reminder timing. Cards collapse to save space. This screen carries context from S-01 through S-04.

Screen structure
Rule Shell pattern
Fixed top: progress bar at 60% (step 3 of 5)
Expanded: scrollable review cards
Fixed bottom: sticky CTA "Looks good"

Back button: ArrowLeft (horizontal line + arrowhead), 44x44px tap target. Returns to previous onboarding step.
Note Context carry-over
Items shown here come from whichever path the user took: voice (S-02), browse (S-03), or search (S-04). This screen is path-agnostic. It shows whatever items were collected.
Card interactions
Interaction Expand / collapse
Tap card header to expand or collapse.
  • Expanded: shows all tasks, "last done" picker, scan button
  • Collapsed: shows task count + "tap to review" hint + chevron
First card auto-expands on screen load so the user sees what a review card contains. Subsequent cards start collapsed.

Animation: 250ms ease-in-out height transition.
Interaction Scan button
Scan icon (PhosphorIcons.scan) triggers the AI camera.
The user photographs a label or device to detect brand and model. Result updates the item name and may add brand-specific tasks.

Scan is per-item, not global. Only appears on expanded cards.
Free tier: 3 AI scans/month. Pro: unlimited.
Interaction Remove button
× button removes the item from the review list.
Triggers a confirmation toast: "Boiler removed" with undo link (2.5s visible, 250ms fade-out).
Undo toast style: cream bg, ink text, plum-lt dot.
"Last done" picker
Pattern Reminder timing
Three-option picker controls when the first reminder fires:
  • Recently: delays first reminder 6 to 8 weeks (user just did it)
  • A while ago: schedules sooner (overdue or close to due)
  • Not sure (default): schedules first reminder soon, conservative approach
"Not sure" is pre-selected. Single-select, tap to switch. No multi-select.

This replaces a date picker. Most users cannot remember exact dates, and precision does not matter at this stage.
Upsell and limits
Note Pro task lock
Free tier: 3 tasks per item. The 4th task (if present) is shown greyed out with a "Pro" pill.
Not blocking, just a visibility nudge. User can still proceed. Tapping the locked task opens the Pro upgrade sheet (S-22).
Note Item counter banner
Shows "X of 10 free items" with a fill bar when the user has 5+ items. Not shown below 5.
At 10 items, the "+ Add another item" link becomes "Upgrade to add more" (colorCta text).
Conversion trigger: free tier caps at 10 items.
Other elements
Interaction + Add another item
Tapping opens an add-item flow (inline voice/text input or quick-browse sheet). New item appears at the bottom of the review list, auto-expanded.

This ensures users who want to add items beyond the initial path can do so without going back.
Rule Card spacing
Individual white cards on cream background. 6px gap between cards. Cream gap IS the visual separator.
No border-bottom separators. No single container wrapping all cards.
Note CTA behaviour
"Looks good" navigates to S-06 (auth). Enabled as long as at least 1 item exists. If all items are removed, CTA changes to disabled state with text "Add at least one item".

Pressing: bg darkens, scale 0.97, 100ms ease-out.
