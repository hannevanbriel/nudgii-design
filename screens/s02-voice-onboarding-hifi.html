Shell + navigation
Rule Progress bar — 40%
S-02 is step 2 of 5 = 40% fill. Full-width edge-to-edge, 3px height, outside SafeArea. colorCta #9B7FD4 fill. Track rgba(26,22,18,0.08). Flutter: LinearProgressIndicator inside ClipRRect(borderRadius:100px).
Rule No screen title
There is no heading above the conversation. nudgii's first bubble IS the opening. No "Step 2 of 5" label anywhere — the progress bar handles that. Back button only.
Rule Back button — arrowLeft SVG
32×32px container, surface bg, 9px border-radius, 0.5px border. SVG: viewBox="0 0 12 12", paths d="M10 6H2" and d="M5 3L2 6L5 9". Minimum tap target wrapped to 44×44px in Flutter with SizedBox.
Conversation bubbles
Rule Sender label
Always lowercase "nudgii" — never "NUDGII". No text-transform. 8px DM Sans 500, letter-spacing 0.4px, colorMidAccessible #6B6358. nudgii is always lowercase everywhere: in copy, labels, code, and documentation.
nudgii bubbles
Left-aligned. Surface white bg, 0.5px border rgba(26,22,18,0.08). Border-radius: 12px 12px 12px 4px (bottom-left flat = "tail"). Max-width 88%. Box-shadow 0 1px 3px rgba(26,22,18,0.04).
User bubbles
Right-aligned. colorCta #9B7FD4 bg, colorCtaFg #FAF8FF text. Border-radius: 12px 12px 4px 12px (bottom-right flat). Max-width 84%.
Confirmed chips
Inside nudgii's confirmation bubble. sage-lt #EAF3DE bg, sage #3B6D11 text, checkmark SVG icon 9px. flex-wrap row with 4px gap. Appear immediately after nudgii confirms the item — not in a separate message.
State-specific decisions
State 1a S-01 context carry-over
If the user selected items on S-01 (tap-to-select preview items), those items pre-populate as confirmed chips in nudgii's opening bubble. nudgii acknowledges them, then asks what else. Suggestion chips exclude already-confirmed items.
State 1b Cold-start hint
No context. nudgii opens with a discovery question. "2–3 items is plenty to start." appears as italic hint below the bubble text in colorMidAccessible. Removes the freeze response from users who think they must add everything.
State 2 Waveform placement
Waveform appears inside the conversation thread, not overlaid on the input bar. The in-progress transcript fades in at 0.65 opacity as a user bubble. Input bar border becomes rgba(196,80,58,0.30). Mic circle turns colorTerra #C4503A.
State 3 One follow-up rule
nudgii asks at most one clarifying question per turn. Confirmed chips appear first, then the question. Suggestion chips update to only show items not yet added. Italic em on the question text highlights it as the active ask.
Input bar
Rule Scan icon
Four corner-bracket SVG (not a camera). Represents the AI scan entry point. PhosphorIcons.scan in Flutter, 14px, stroke colorMidAccessible stroke-width 1.5.
Rule Privacy note — always visible
7px italic DM Sans, rgba(107,99,88,0.45). Always present below the input bar. Copy: "audio sent for transcription only · deleted after · responds in text". Centers reassurance at the exact moment the user decides to tap the mic.

4 states defined: S-01 context pre-loaded (1a), fresh start (1b), listening (2), after first turn (3).
No screen title. nudgii bubble is the opening. Progress bar at 40% fill. No "Step X of Y" label anywhere.

Privacy note: always visible below input bar. 9px italic, rgba(107,99,88,0.45). Not dismissable.
S-01 context (State 1a): items selected on S-01 appear as pre-confirmed sage chips in nudgii's first message. Suggestion chips adapt to exclude already-confirmed items.
Waveform (State 2): 9 bars inside conversation thread, not overlaid on input bar. colorCta fill. "Listening..." italic label. Partial transcript fades in at 0.65 opacity. Mic circle: colorTerra. Input bar border: rgba(196,80,58,0.30).
Follow-up rule (State 3): max one clarifying question per turn. Confirmed chips first, then question, then updated suggestion chips. Italic em on the question.
