S-02 conversation pattern

Three design proposals for the voice onboarding conversation: the existing lo-fi pattern as reference, and two new hi-fi directions. Review and decide before hi-fi production starts.

What works keep
  • Chat thread layout — feels like a familiar conversation. Users understand it immediately (WhatsApp/iMessage pattern).
  • Bubble contrast — nudgii left (white), user right (purple). Direction signals who said what without labels.
  • Follow-up question — single question per turn. Respects attention without overwhelming.
  • Italic emphasis — key question highlighted in CTA color draws the eye to what needs answering.
What needs updating fix
  • Floating mic — disconnected from the conversation. Visually ambiguous: is it a button or decorative?
  • No input bar — "or type below" refers to nothing visible. Users who want to type have no affordance.
  • No scan option — scan is available at S-05 but not here. A missed opportunity for confident AI capture.
  • No privacy signal — audio is transcription-only and deleted. Users don't know this. Trust gap.
  • Dutch copy — all design files must use English copy.
Option A — recommended
Chat thread + persistent input bar
Keeps the conversational chat bubble pattern from the lo-fi — users already understand it. Replaces the floating mic with a persistent input bar at the bottom: scan icon left, text placeholder center, mic button right. Three states: empty, listening, and after a turn.

What's new in Option A new
  • Persistent input bar — scan icon left, text input center, mic right. Always visible, always accessible. Removes the floating-mic ambiguity.
  • Active mic state — mic turns terra red during recording. Input bar border changes color. Unmistakable recording signal.
  • Confirmed items chips — sage-lt chips inside nudgii's confirmation message. User sees exactly what was captured without reading prose. Fast to scan, easy to correct.
  • Contextual suggestions — suggestion chips update after each turn to reflect what's likely next given what was captured. "You added 2 cars, here's winter tyres."
  • Privacy note — always visible below the input bar. Builds trust. Key for GDPR in BE/NL.
What stays from the original keep
  • Chat thread layout with bubbles (the part users liked)
  • nudgii left / user right positioning
  • Italic emphasis on follow-up question
  • Single question per turn (max 1 clarifying question)
  • Suggestion chips for cold-start help
Option B — alternative
Contextual feed — no chat thread
The alternative direction from the new mockup: no conversation thread visible. nudgii's responses appear as confirmation chips. Cleaner, more minimal. But loses the conversational warmth and makes multi-turn harder to follow.

Pros of Option B
  • Cleaner first impression for new users who might feel intimidated by a chat interface
  • Added items are immediately visible as a list, no hunting through conversation bubbles
  • Easier to scale to many items without the screen filling up with chat bubbles
Cons of Option B
  • Loses conversational warmth. nudgii's follow-up question becomes a "nudgii asks" chip, not a natural part of dialogue.
  • Mixed metaphors. Voice input + item card list is two different UI paradigms on one screen.
  • Worse for multi-turn. After 3+ turns, the screen looks like a task list not a conversation. Users don't know when they're "done".
  • nudgii's personality disappears. "Got it." and the italic question are nudgii's voice. Option B loses all of that.

Design recommendation
Option A: Chat thread + persistent input bar
Option A keeps what users already liked in the original (warm conversational bubbles, nudgii's voice) while fixing the three real issues: the floating mic with no text alternative, the missing privacy note, and the absent scan affordance. It also adds confirmed-item chips so users see exactly what was captured without parsing prose. Option B is cleaner but trades warmth for minimalism, and nudgii's personality is one of the strongest things the app has going for it in onboarding.
