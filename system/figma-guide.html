nudgii · Figma setup guide · March 2026

From zero to
handoff-ready.

Five phases. Each one builds on the last. Follow in order — skipping Phase 1 makes everything else harder. Total time: roughly 4–5 hours for a solo developer starting from scratch.

01
Set up tokens
Before any frames. This is the foundation — every component and screen references these tokens. Change a token here and it updates everywhere automatically.
⏱ ~45 minutes · do this first, never skip it
1
Install Tokens Studio plugin plugin
In Figma: Main menu → Plugins → Browse plugins → search "Tokens Studio for Figma" → Install.

Free tier is sufficient for nudgii. You don't need the Pro tier unless you want GitHub sync (useful later when you have a repo set up).
2
Import the nudgii tokens JSON 2 minutes
Open Tokens Studio → click the Settings tab → Import → paste the contents of nudgii-tokens.json (the file included alongside this guide).

This imports all color, spacing, radius, and typography tokens in one step. You should see four token sets appear: nudgii/color, nudgii/category, nudgii/dark, nudgii/spacing, nudgii/radius, nudgii/typography.

Enable the light mode tokens as active: click the checkbox next to nudgii/color, nudgii/category, nudgii/spacing, nudgii/radius. Leave nudgii/dark unchecked for now — you'll enable it when working on dark mode screens.
3
Convert tokens to Figma Variables
In Tokens Studio → Sync → Export to Figma Variables. This creates native Figma Variables collections that components can reference directly — no plugin required for other designers reading the file.

After export, open Local Variables (top right in Figma, or Edit → Variables). You should see:
  • nudgii/color — 13 color variables
  • nudgii/category — 10 category variables
  • nudgii/dark — 8 dark mode variables
  • nudgii/spacing — 8 number variables
  • nudgii/radius — 5 number variables

Set up modes on the color collection: rename "Default" to "Light", then duplicate it and name it "Dark". In the Dark mode column, update each variable value to the equivalent from nudgii/dark tokens.

02
Create the file structure
Four design files plus one component library. Never put everything in one file — Figma gets slow past 40 frames and the performance degrades noticeably on an M-series Mac.
⏱ ~15 minutes · naming is the whole point here
📚
nudgii · Components
The component library. Publish as Team Library. Every other file imports from here. This is the only file where components are built — never build components inside a screen file.

5
Frame naming convention — use this exactly
Frame names become Flutter route names. Name them as paths:

welcome → path-a/voice → path-a/transcript → path-a/review → dashboard/active

Inside each frame, layer naming matters too. Use: component/variant/state
Example: nudge-card/home/overdue · button/cta/default · tab-bar/today-active

Never leave a layer called "Frame 47" or "Rectangle 3". Name every layer before moving to the next frame. This sounds tedious — it saves hours when you're searching for something in a 30-frame file.
03
Build the component library
Build in nudgii · Components. Publish. Then never touch them inside a screen file. Each component spec below tells you exactly what variants and properties to create.
⏱ ~2–3 hours · this is the most valuable time you'll spend
Nudge card
Properties:
· category — home / vehicle / garden / subscriptions / health
· status — overdue / today / upcoming / ok
· done — boolean (strikethrough + opacity)

Auto-layout: horizontal, 10px gap, 10px 12px padding.
Status changes only the date label color token — no other changes. Category changes the icon and its bg token.
Button
Variants: primary / secondary / ghost
Size: default (50px) / sm (40px)
With icon: boolean — shows mic icon slot

Primary: apricot-dk fill, cream text, pill radius.
Secondary: transparent, border-md stroke.
Ghost: transparent, no border, mid text.
Category icon chip
Property: category — home / vehicle / garden / subscriptions / health

Fixed 30×30px, radius 9px. Uses cat/[name] token for stroke and cat/[name]-bg token for fill. Icon SVG swaps per variant.
Filter chip
State: default / active
Category: all / home / vehicle / garden / subscriptions

Default: transparent bg, border-md. Active: ink bg, cream text. Category chips show a 4px dot in category color when not "All".
Tab bar
Active tab: today / ask / you

Fixed height: 52px (+ bottom safe area). Three items, space-around. Active: ink icon + label + 3px apricot-dk pip below. Inactive: border color 22% opacity.
Done overlay
Text: title (string) and sub (string) as properties

Full screen, plum bg. Center column: ii mark (40px) → serif title → italic sub → undo button. In Figma prototype: use Smart Animate, scale + opacity, 400ms, spring easing. This is the only theatrical animation.
Pro badge
Full pill: plum-lt dot + "nudgii Pro" text, 10% opacity bg, 20% border. 10px font, 500 weight.

Used only on the profile screen header. Never on task cards, filter chips, or navigation.
Streak row
Property: count (number)

Horizontal card, white bg. Left: star icon in apricot-dk. Center: "n-week streak" (500) + "your stuff is quietly grateful" (italic, mid). Right: DM Serif Display number in apricot-dk.
6
Publish the library do this before Phase 4
In nudgii · Components file: Main menu → Libraries → Publish. Add a description: "nudgii component library v1 · March 2026".

In each of the four screen files: Main menu → Libraries → nudgii · Components → Enable.

Important: whenever you update a component in the library, re-publish. Screen files will show an "Update available" notification — accept it. Never edit a component instance inside a screen file — always go to the source component.
04
Build the screens
Now that tokens and components exist, each screen is mostly assembly. Use the HTML reference files — screenshot at 2x, place as a locked reference layer, build on top.
⏱ ~20–30 min per screen · do welcome and dashboard first
7
Frame setup for every screen
Use iPhone 16 Pro frame preset (393 × 852px) or iPhone 14 (390 × 844px) — pick one and use it for all screens. Never mix frame sizes in the same file.

Layer order in every frame (top to bottom in the layers panel):
  • Status bar (auto-layout, always topmost)
  • Header zone (screen-specific)
  • Scrollable content (auto-layout, vertical, stretches)
  • Tab bar (pinned to bottom with constraints)
  • Done overlay (hidden by default, shown in prototype)
  • Reference (locked image, lowest layer)
8
Welcome screen — build this first
Frame name: welcome

Layer structure:
  • status-bar — time left, battery right, 11px DM Sans 500, ink 36% opacity
  • header — auto-layout vertical, 13px gap, 14px 20px padding
    • top-row — ii mark (20px) left, free badge right, space-between
    • headline — text style nudgii/display/welcome, ink
    • subline — 12px DM Sans 300 italic, mid
  • preview-cards — 3 instances of nudge-card component, status overridden per locale
  • path-divider — line + "Hoe wil je starten?" label
  • paths — button/primary (Path A) + hint + button/secondary (Path B) + hint + button/ghost (Path C)
9
Dashboard — two frames (active + empty)
Frame names: dashboard/active-light and dashboard/empty-light

Header zone — no gradient, flat cream:
  • eye — 9px DM Sans 500 uppercase, mid
  • greeting — text style nudgii/display/greeting. Name text is italic em in apricot (not apricot-dk here)
  • context — 11px italic, mid 32% opacity
  • actions — search button (28×28, mid-opacity bg) + add button (28×28, ink bg)

Filter chips row — horizontal auto-layout, 6px gap, 14px top padding.

Content sections — "Overdue" section label + overdue cards. "This week" section label + cards. "Coming up" label + cards.

Empty state — center the ii mark (tiny, 14% opacity) + DM Serif Display 22px "Nothing overdue. Suspicious." + 12px italic subtext. Below: "Done this week" section with 30% opacity done cards.
05
Wire the prototype
Connect the onboarding flow first — that's what you'll user test. Get 2–3 people to tap through it before building any Flutter code.
⏱ ~1 hour for onboarding flow · 30 min for daily use
A Onboarding · Path A (voice) — the primary flow
welcome → path-a/voice → path-a/transcript → path-a/ai-response → review → completion → dashboard/active-light
Trigger: tap Path A button on welcome. Transition: push right (300ms ease-in-out) for all except completion → dashboard which uses a custom fade (250ms).
B Onboarding · Path B (browse) — secondary flow
welcome → path-b/browse → review → completion → dashboard/active-light
Trigger: tap Path B button on welcome. Same transitions as Path A from review onward.
C Onboarding · Path C (direct search)
welcome → path-c/search → review → dashboard/active-light
Trigger: tap ghost link on welcome.
✓ Daily use · Done overlay — the key moment
dashboard/active-light tap card → dashboard/done-overlay 4s or undo → dashboard/active-light
Transition: Smart Animate, 400ms, spring easing cubic-bezier(0.34, 1.56, 0.64, 1). The overlay must scale from 0.65 to 1 — set the overlay frame to 65% scale in the default state, 100% in the triggered state. Figma Smart Animate handles the interpolation. This animation is the most important thing in the prototype to get right.
10
Share and test before building Flutter critical
Share the prototype link (Present mode → Share) with 2–3 people who haven't seen nudgii before. Watch them tap through the onboarding. Don't explain anything — observe silently.

What to watch for:
  • Do they tap Path A without hesitation? If they pause, the button hierarchy isn't clear enough.
  • Do they understand the three locale preview cards are examples, not interactive? If they tap them expecting something, add a subtle label.
  • Does the done overlay feel earned or startling? Startling means the transition is too fast.
  • Do they find the Path C ghost link? They shouldn't need to — but they should be able to if they look.

Fix what you see in Figma before writing a line of Flutter code. Changes in Figma take minutes. Changes in Flutter take hours.
