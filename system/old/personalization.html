Personalisatie

Hoe nudgii bepaalt wat het toont — locale, templates, AI-output, gebruikersregels.

Mechanismen
Vier mechanismen
1 · Locale-regels (device)
Stille locale-detectie
Device locale wordt silently gelezen bij installatie — geen vraag aan gebruiker.
Bepaalt: welke items S-01 toont, welke taal de digest gebruikt, welke seizoenslogica actief is.
LocaleStandaard items S-01
nl-BEVaatwasser · Winterbanden · Olijfboom
nl-NLCv-ketel · Winterbanden · Tuin
fr-BELave-vaisselle · Pneus hiver · Olivier
de-DEHeizung · Winterreifen · Garten (v1.1)
Regel: NOOIT locale-content voor het verkeerde land tonen. Locale bepaalt ook seizoenslogica, kennisbank-taal en digest-taal.
2 · Templates (pre-loaded)
nudgii's moat — 30+ templates
30+ templates shipped with the app. Elke template bevat: naam, categorie, standaard-interval, why-it-matters copy, seizoenslogica.
Voorbeelden:
Vaatwasser zout — 6–8 weken
Cv-ketel service — 12 maanden
Winterbanden — seizoensgebonden (voor november)
Olijfboom snoei — feb + aug (BE/NL/FR)
Templates zijn startpunten — altijd overschrijfbaar door gebruiker.
Templates leven in Supabase. PM beheert via dashboard — geen app release nodig voor updates.
3 · AI-output (Claude API)
Eén JSON-call, volledig reviewbaar
Claude API — claude-sonnet-4-6 — single JSON call.
Returns: item_type, brand, model, tasks[].
Context altijd meegestuurd: locale + today's date + last-maintenance waarde.
Voorbeelden:
"4 airco's" → Claude stelt verduidelijkingsvraag (max 1)
"Cv-ketel en vaatwasser" → templates laden direct
AI-output is altijd reviewbaar op S-05 voor opslaan. Nooit stilletjes opgeslagen.
4 · Gebruikersregels (live)
Post-onboarding CRUD
Na onboarding kan de gebruiker alles aanpassen via Supabase-backed CRUD:
· Interval aanpassen (per taak)
· Holiday mode activeren
· Taak snoozen (met reden)
· Custom item toevoegen
· Regio corrigeren (S-24)
Alle regels worden lokaal gecached voor offline gebruik. Sync bij reconnect.
