Post-onboarding add item flow

What happens when a user adds an item after onboarding, via the FAB. All three input paths (Browse, Scan, Tell me) converge on the same Add Item Review screen before the item is created.

Spec Design needed v1.0 · April 2026

Flow overview

1
FAB tap

↓

Browse (S-03 pattern, single-select): category grid, search, tap an item template.

Add Item Review screen

This is the key new screen. It shows the user what nudgii plans to create before they commit. Like S-04 Review during onboarding, but for a single item post-onboarding.

Screen structure

Hero zone
Category wash background (full width, 120px height). Category icon (44px circle, white overlay). Item name in DM Serif Display, editable (tap to rename). Category label below. If AI scan: brand + model shown. Back arrow top-left.
Sub-type question (if required)
Appears for items with subtype_required: true (Car, Boiler, Oven, Garden pond). Task list does NOT render until user picks. See "Sub-type question pattern" section below for the full UX.
Task preview
"nudgii found X tasks" summary line. Task cards (collapsed, same style as S-17). Each shows: task name, cadence, next due date. Tappable to preview "Why it matters." Removable (X button, with confirm). "Add custom task" ghost link at bottom. For sub-typed items: tasks filter to the chosen sub-type.
First nudge preview
"Your first nudge" highlight card. Shows the earliest task: name + "in X days/weeks." Apricot wash background. Communicates when nudgii will first reach out. Builds trust: "I know when to remind you."
Detail fields (optional)
Collapsible "Add details" section. Fields vary by item type (see item-templates.html). Brand and model pre-filled if from AI scan. All optional at this stage. User can fill in later on S-17.
Nickname suggestion
If item type has a personality name suggestion: show as a chip below the item name. "Want to call it Bubs?" with tap-to-accept. Dismissible. One-time offer. After AI scan: brand-based suggestion possible ("The Bosch").
Sticky bottom: CTA
"Add to my stuff" plum CTA button. Full width. Sticky to bottom (same SafeArea pattern as onboarding). Below: "Edit tasks" ghost link for users who want to customize before adding.

Sub-type question pattern

Some items (Car, Boiler, Oven, Garden pond) need a sub-type question before tasks generate, because the wrong sub-type produces wrong tasks that destroy trust. The UX pattern differs by how likely the user is to know the answer.

Pattern A — Force pick (Car, Oven, Garden pond)

When users always know

Car fuel type, oven gas/electric, pond fish/decorative. Users know these with near-100% confidence. No escape hatch needed.

Chip picker
Label: the subtype_question string ("What does it run on?" / "What type?" / "What's in it?"). Below: horizontal chips, 2-5 options, sentence case. Tap once to select. Task list renders immediately after selection. Optional one-line help hint for edge cases (Oven: "Electric = glows and heats. Gas = has a flame.").
No "Not sure" option
Removing this option forces a decision users can make. Trying to be too flexible creates quiet failure (users pick "not sure" out of laziness, then get wrong tasks forever).

Pattern B — Visual picker + Photo help + Deferred (Boiler)

When users often don't know

Renters, recent buyers, inherited homes — many users genuinely don't know their boiler type. Three layers, in order of user effort:

Layer 1 — Visual picker (primary)
4 CARDS (not chips): Gas combi / Gas system / Electric / Oil. Each card has a simple illustration showing the silhouette of that boiler type (they look quite different). Plus a hint line: "Most common in BE: Gas combi." Users with any familiarity pick in 3 seconds.
Layer 2 — Photo help (fallback)
Secondary link below the cards: "I don't know what type I have." Opens a help card: "Look for the info plate on the front of the unit. It usually says the type and brand. Want to snap a photo and I'll identify it?" Photo flow reuses AI scan infrastructure (Claude Vision reads the plate). This is the best real-world escape — the info IS physically available.
Layer 3 — Deferred (last resort)
Tiny ghost link below photo CTA: "Add anyway, I'll figure this out later." Creates item with UNIVERSAL tasks only (annual service + filter clean — tasks that apply regardless of type). Item shows "Setup incomplete" chip on My Stuff (cream/apricot, dismissible). Today tab gets ONE gentle card (dismissible, no weekly nag): "Finish setting up your boiler so I can nudge you about the right things." When user eventually picks type, type-specific tasks get added and the chip disappears.
Why 3 layers work
Honest about the gap. No guilt. Functional in the meantime. Clear path to completion. Respects users who don't have physical access right now but still want to set things up.

AI scan complement

When users add via scan, sub-type is often pre-filled

Claude Vision can identify sub-type from a photo in many cases:

  • Car fuel type: fuel cap color, badge, charging port, exhaust. Usually accurate.
  • Boiler type: info plate reading. Very accurate when plate is visible.
  • Oven: visible burners (gas) vs just elements (electric). Usually accurate.
  • Garden pond: fish visible or not. Sometimes ambiguous (fish may be hiding).

If AI confidence is high (>0.85), sub-type is pre-selected in the chip picker. User can override. If AI confidence is low, chip picker appears empty and user picks manually.

Data model (Arne's call)

Two options documented in item-templates.html Sub-types section. Both options support the same UX. AI generation and curated templates use the same schema. Arne picks at implementation.

Item limit behavior (item 11+)

Free tier: 10 items max

When adding item 11+, the Add Item Review screen shows a soft inline message between the task preview and the CTA:

"This will be your 11th item. Free plan covers 10."

Below the message: "Go Pro" link (not a button, not a blocker).

The item still gets added. It appears on My Stuff in paused state (muted, "Pro" pill, no reminders sent). User chooses which 10 stay active, or upgrades. Never block. Never guilt. Never hide data.

Template resolution: curated vs AI-generated

Every item goes through a template lookup before reaching the Add Item Review screen. Two paths:

Curated template match

Item type exists in the curated library (see item-templates.html). Tasks, cadences, why/how content are pre-defined. Instant, no API call needed. Source badge: none (default experience).

No match: AI-generated template

Item type not in curated library. Claude API generates tasks on the fly (1-3 second loading state). Tasks appear on Add Item Review with "Tasks suggested by AI" badge. User reviews, edits, removes, or adds tasks before confirming. Full spec: ai-template-generation.html.

Path-specific behavior

Browse path (FAB > Browse)

  • Opens S-03 pattern in single-select mode (not multi-select like onboarding)
  • User taps one item from the template library
  • If curated template exists: transitions to Add Item Review with template data pre-filled
  • If no template match (user searched for something custom): AI generates template, transitions to Add Item Review with "AI-suggested" badge
  • Category auto-assigned from template or AI

Scan path (FAB > Scan)

  • Camera opens immediately (no intermediate screen)
  • User photographs the item (appliance label, car dashboard, plant leaf)
  • Claude Vision API returns: item_type, brand, model, tasks[]
  • Transitions to Add Item Review with scan results pre-filled
  • Brand + model shown in hero zone. "Identified by AI scan" source badge.
  • If scan fails: error toast + "Try again" or "Add manually" options
  • Free tier: 3 scans/month. Counter shown before scan.

Tell me path (FAB > Tell me)

  • Opens conversation UI (S-02 pattern)
  • User describes what they have via voice or text
  • nudgii identifies the item and suggests tasks
  • When user confirms, transitions to Add Item Review
  • If item type is unknown: nudgii asks for category, creates item with zero template tasks
  • Free tier: 3 voice sessions/month.

After adding: My Stuff integration

  • New item appears at the top of the appropriate status group ("On track" for new items with all future tasks)
  • "New" badge: subtle category wash pulse animation on the item card, fades after first view (600ms pulse, 3 cycles, then static)
  • Tasks immediately appear on Today tab if any are due within the current window
  • Plan tab shows the item's tasks projected across future months
  • First push notification for this item: scheduled based on earliest task due date

Edge cases

Duplicate item

If user adds an item that matches an existing item (same type + similar brand/model): Add Item Review shows amber inline warning: "You already have a Bosch dishwasher. Add another?" with "Yes, add" and "Cancel" options. Allows duplicates (some homes have two dishwashers).

Unknown item type

No template match. Add Item Review shows: item name (user-provided), category picker chips, zero tasks. "I don't know this one yet. Want to add a task?" with "Add task" CTA. User can create custom tasks with their own cadence.

Scan failure

Camera returns no match or low confidence. Toast: "Couldn't identify this item." Options: "Try again" (re-scan), "Tell me instead" (switch to conversation), "Add manually" (empty Add Item Review with category picker).

Cancel mid-flow

Back button on Add Item Review: returns to previous screen (Browse/Camera/Conversation). No item created. No data lost (can resume). No "are you sure" dialog (lightweight, not a form submission).
