Brand & logo

nudgii is always lowercase. The ii lettermark is the icon. These rules do not bend.

Brand name locked forever
nudgii — always all lowercase, everywhere. App UI, marketing, code comments, this document. Never "Nudgii." Never "NUDGII." The double-i is the brand's typographic signature — capitalising it erases the point.

Only exception: OS-forced capitalisation in system fields (App Store title field etc.) — this is a system constraint, not a brand choice.
ii lettermark — welcome screen spec locked v2.2
20px height. Left-aligned. Inline with the free badge. Not centered. Not a hero.

The mark mirrors the icon exactly — same three colors in miniature:

Left pill: cream #F5F0E8 with ink outline (mirrors icon left cream pill)
Right pill: plum-lt #9B7FD4 at 55% opacity (mirrors icon right pill — also the CTA color)
Dot: apricot #E8A87C top-right (mirrors icon dot — warm accent)

This is why the welcome screen now coheres with the homescreen icon. The ii mark, the CTA button, and the icon right pill are all #9B7FD4 — one color, three moments.

Primary CTA — light plum updated v2.2
All primary action buttons use #9B7FD4 background with #FAF8FF lavender-cream text. This applies to: welcome screen Path A button, onboarding confirm buttons, dashboard add button, active filter chip, tab bar active pip.

Contrast ratio: 4.7:1 on cream (passes WCAG AA). Box shadow: 0 2px 12px rgba(155,127,212,0.28) — subtle plum glow that ties the button to the icon.

Why light plum: the icon right pill, the CTA button, and the Pro badge are all #9B7FD4. The done overlay is #2D1F4A — same family, 3× deeper. Tapping the button feels like a preview of the reward. The color thread: icon → CTA → overlay.

Apricot-dk #C8784A is now status-only — due-soon date labels. Never used on interactive elements.

Dark mode rules
Default is light. Dark follows system preference (iOS dark mode toggle). Both modes must be fully designed — "just invert" is not acceptable.

The profile/settings screen uses an ink band header in both modes. The done overlay uses plum #2D1F4A in both modes — it's already dark enough to read as a mode shift in light mode, and stays consistent in dark.

CTA button color is identical in both modes — #9B7FD4. Light plum works against both cream and dark-card backgrounds.

Category color rules locked
Category colors appear only in the icon chip (30–34px rounded square). Never in: date labels, section headers, card backgrounds, tab bar, filter chip backgrounds, or anywhere carrying status meaning.

Adding a new category: pick one muted hue not already in the system. Add --c-name: #hex and --c-name-bg: rgba(r,g,b,0.10). One icon SVG. Done — the system absorbs it without redesign.

10

Icon system

Phosphor Icons · regular weight · outline only throughout. One library, one weight, no exceptions.

Library & style locked
Library: Phosphor Icons — six weights available, nudgii uses regular weight only.
Style: outline throughout — no filled icons anywhere in the app. Consistent with the restrained, editorial personality of the design system.

Flutter package: flutter_phosphor_icons. Figma: Phosphor Icons plugin (free).
Sizes by zone locked
12px — inside category chip (task cards, welcome preview cards)
14px — action buttons (search, settings, camera, close)
16px — mic icon inside CTA button
19–20px — tab bar icons

Never mix sizes within the same zone. Never use an icon outside its defined zone size.
Stroke width rules
stroke-width: 1.5 — all outline icons at 14–20px
stroke-width: 1.8 — mic icon inside CTA button (needs slightly more presence)
stroke-width: 1.8 — icons at 12px in category chips (compensates for small size on cream)

Never use weight below 1.5 — disappears on cream at small sizes.
Color rules by zone
Tab bar active: ink #1A1612
Tab bar inactive: ink at 18% opacity
Action button on ghost bg: mid #8A8070
Action button on CTA bg: cta-fg #FAF8FF
Category chip icons: category color token only (e.g. #C87850 for home)
Status icons in pills: status color token (terra / garden / mid)

Never use category colors on non-category icons. The category color system is separate from the UI icon color system.
Revised icons — 3 changes updated v2.3
Settings: Faders — three vertical sliders. Reads as preferences/personalisation, not mechanical configuration. Replaces GearSix.

Camera / AI scan: Scan — four corner brackets with a dashed scan line. Universally understood as "scan this object." Replaces CameraPlus which implied photography.

Edit / rename: PencilLine — pencil with a baseline. Reads as "edit text content." Replaces PencilSimple which was too generic.
Tab bar — floating pill updated v2.3
nudgii uses a floating pill tab bar — a white rounded card that hovers above screen content with a shadow. No hard border-top edge. Content scrolls behind it.

Spec: background white · border-radius 22px · padding 9px 4px 11px · shadow 0 4px 20px rgba(26,22,18,0.11) · border 1px solid rgba(26,22,18,0.07) · bottom 14px from screen edge.

Active tab: full ink + 3px cta pip below label. Inactive: ink 20% opacity.
FAB — floating action button updated v2.3
Position: bottom-right, adjacent to the tab pill. Never inside the pill.
Shape: rounded square 46×46px, border-radius 15px. Not a circle — matches the pill's visual language.
Color: cta #9B7FD4 background, cta-fg #FAF8FF Plus icon. Shadow 0 4px 16px rgba(155,127,212,0.35).
On tap: Plus rotates to × (45° transform, spring easing). Scrim dims content to rgba(26,22,18,0.18). Action sheet animates up from bottom-right.

Action sheet order — ergonomics first:
↑ Top (furthest reach) — Ik weet wat ik nodig heb · direct search · power users
· Middle — Laat me zien · browse categories
↓ Bottom (nearest thumb) — Vertel het me · voice · primary path

Voice is at the bottom because it is the primary action and the thumb naturally rests nearest the FAB after tapping it.
Touch targets never smaller
Minimum tap target: 44×44px always — Apple HIG requirement, no exceptions.
The visual icon can be 14–20px. The tappable area must be 44×44px minimum.
In Flutter: wrap icon buttons in SizedBox(width: 44, height: 44) with centered icon, or use IconButton which handles this automatically.
never What not to do
Never mix Phosphor with any other icon library (SF Symbols, Material, Lucide) in the same screen.
Never use a filled icon — outline only, always.
Never resize icons outside the defined zone sizes.
Never use an icon purely for decoration — every icon must carry meaning.
Never use category colors on tab bar or action button icons.
Never use GearSix, CameraPlus, or PencilSimple — replaced in v2.3.

Figma workflow

Solo developer setup. Fast, consistent, handoff-ready without becoming a second job.

01
Set up Variables first
Before any frames. Create two collections:
Color: color/cream, color/ink, color/apricot-dk, color/terra, color/plum, color/plum-lt etc.
Category: cat/home, cat/home-bg etc.

30 minutes once. Saves hours of find-and-replace.
02
One Components file
File: nudgii · Components. Publish as Team Library. Components: nudge card, filter chip, button (3 variants), tab bar, done overlay, category icon, section label, streak row, pro badge.

Every screen file uses these — change once, updates everywhere.
03
One file per flow
File 1: Onboarding · File 2: Dashboard · File 3: Item + task detail · File 4: Profile + settings.

Frame names = Flutter route names. welcome/path-a-btn not "Button 3".
04
HTML files as reference
Screenshot each HTML file at 2x. Place as "Reference" layer in Figma. Rebuild on top using components + variables.

20–30 min per screen. Rebuilding is also a design review.
05
Dev Mode + Tokens Studio
Figma Dev Mode (free) → exact spacing + properties for Flutter.
Tokens Studio plugin → syncs Figma variables to JSON in your repo. Variable names match Flutter token names exactly: colorInk, colorApricotDk.
06
Prototype the overlay first
Wire the done overlay with Smart Animate, scale + opacity, 400ms, spring easing. Show to 2–3 people before writing any Flutter code. Watch where they hesitate — that's where the design has a question mark.
Don't do this
Don't design directly in the handoff file. Design in "Working" page, copy finished frames to "Handoff" page. Prevents accidental edits to locked screens.
The one hour that saves ten
Name every layer before building the next screen. "Frame 47" is useless. dashboard/nudge-card/overdue is searchable, readable, and maps to Flutter.

Locked rules

These don't get questioned in design reviews. They are decided.

never Gradient anywhere
No gradient backgrounds, headers, or CTAs. Warmth comes from cream + apricot accent + typography weight. Gradients read as tacky at the quality level nudgii targets.
never Left-border accent lines
Colored left stripe on cards is the most overused pattern in AI-generated UI. Status lives in the date label color only. No exceptions.
never Plum outside done overlay
Deep plum #2D1F4A appears only in the done overlay background. The moment it appears on a chip, header, or badge the overlay loses its impact.

Light plum #9B7FD4 has two sanctioned uses only: the CTA button (and its echoes — active chip, tab pip, add button) and the Pro badge. It never appears in category icons, card backgrounds, section labels, or any status element.
never Category colors on status
A slate vehicle icon is always slate — overdue or not. The icon tells WHAT, the date tells WHEN. Two independent systems that never overlap.
never Blue-tinted dark bg
Old ink #1E2832 had a blue hue — removed. Dark base is deep sage #111614. No blue, no cold, no developer-tool feeling.
always nudgii lowercase
Always nudgii — never Nudgii, never NUDGII. In all app copy, marketing, documentation, and code comments. The double-i is the brand's typographic signature.
always One serif moment per screen
DM Serif Display appears once per screen. Welcome: headline. Dashboard: greeting. Detail: witty opener. Done overlay: confirmation. Never two visible simultaneously.
always White cards on cream
Task cards and UI cards are white on cream. Subtle but essential — creates depth without contrast. Never use off-white for cards on cream — they merge.
always CTA = light plum updated v2.2
#9B7FD4 on all primary action buttons. Pulled directly from the icon's right pill — the color thread is icon → CTA → done overlay. Lavender-white #FAF8FF text. Never ink, never apricot, never green. Apricot-dk is now status dates only.
always Sentence case everywhere
Buttons, chips, labels, section headers — sentence case. Never Title Case. Never ALL CAPS in user-facing copy. This includes filter chip labels and tab bar labels.
