v0.3 changes — 2026-03-21
  • Path A = voice OR text — same engine. "Vertel het me" opens the conversational AI interface. "Liever typen?" is a first-class equal alternative — same Claude API call, same review outcome. Not a fallback, not secondary.
  • Last maintenance chip default locked. Always "6 maanden geleden" (amber chip). Never "Recent", never "Unknown". Locked on S-03 and S-04. User can update on S-05.
  • S-07 auto-advance removed. Completion screen never auto-advances to dashboard. User must tap "Open mijn schema" consciously. No timer, no countdown.
  • S-05 confirm-only. No inline editing on review screen. Delete (×) per item only. Scan icon (Phosphor Camera) per item row for richer AI output. Deep edits belong post-dashboard.
  • Location screen removed. Region detected silently from device locale. No permission screen, no user decision required at onboarding.
  • OD-01 resolved. Done overlay fires from dashboard swipe only (>40% threshold). Task detail "mark done" = quiet in-screen banner only. S-07 completion = sage check circle — no plum overlay.

v0.3 Screen specs · Edge cases · Notifications
S-06 — Auth (deferred)
User has 60+ seconds invested by S-06 — conversion higher here than at the door.

Layout: ii mark 22px centred · Headline "Bijna klaar." DM Serif 18px · Subline about saving schedule

Auth options: Apple SSO (first — iOS HIG) · Google SSO (second) · Magic link (inline "Check je e-mail" on tap — no new screen)

Auth failure: Retry visible immediately · session state preserved · items never lost
S-08 — Push permission
Shown after S-07, before dashboard. Context is given first — explaining why push matters.

If declined: user continues to dashboard regardless. Not a blocker.

Restore path: S-23 Settings → Notifications
Edge cases — S-26 · S-27
S-26 Mic fails:
Message: "Microfoon werkt niet. Tik hieronder om te typen."
Action: redirect to text input — never a dead-end.

S-27 Claude parse fails:
Message: "Dat was lastig te begrijpen. Laten we het anders proberen."
Action: redirect to S-03 Browse categories.

Both edge states: calm tone, clear action, never alarming.
Notification rules
Weekly digest email Mon 08:00 (user-configurable). Subject must reference specific item.
Due date push Day of due_date at configured time. One nudge — never repeat same day.
Overdue push 1 day after due_date if not done. Terra urgency, honest tone.
Return notification First open after away_mode_until. "Welcome back — here's what piled up."
